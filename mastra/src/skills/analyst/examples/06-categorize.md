# 例 6 · 未分类信号 + 候选分类, 判断归属 (chosen_project_id)

信号未归类时, user prompt 末尾会附【分类归属判断】候选列表. 此时要从候选里选最匹配的一个分类 id 填进 chosen_project_id.

**输入**:

> 宁德时代储能中标了沙特一个 4GWh 的大单, 海外储能放量比预期快.
>
> 【分类归属判断】这条信号尚未归类。请结合信号内容与下列各分类的「分析指引」, 选出最匹配的一个分类, 把它的 id 原样填进 chosen_project_id:
> - 11111111-1111-1111-1111-111111111111 「新能源」: 锂电、光伏、储能、电网侧的产业链与供需拐点
> - 22222222-2222-2222-2222-222222222222 「消费」: 白酒、餐饮、零售、出海消费品
> - 33333333-3333-3333-3333-333333333333 「AI 硬件」: 算力、HBM、服务器、先进封装
> 规则: 只能填上面列出的某个 id; 只有一个候选时直接选它; 信号与所有分类都明显不相关时填 null (不要硬塞)。

**输出** (JSON):

```json
{
  "tags": ["储能", "宁德时代", "海外大单"],
  "related_assets": [
    { "ticker": "宁德时代", "rationale": "海外储能大单直接放量, 储能业务边际改善", "order": "first" },
    { "ticker": "阳光电源", "rationale": "储能 PCS / 系统集成同步受益于海外放量节奏", "order": "second" }
  ],
  "cognitive_layer": "second",
  "consensus_check": "leading",
  "one_line_summary": "海外储能放量超预期, 储能链海外占比拐点",
  "chosen_project_id": "11111111-1111-1111-1111-111111111111"
}
```

**关键点**: 信号讲储能 (属新能源链), 命中「新能源」→ 填其 id. chosen_project_id 必须**原样照抄**候选里的某个 id, 不改写、不编造; 若信号与三个分类都不沾边则填 null.
