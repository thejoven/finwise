# Phase 2 · 仪式 · Overview

> W9-W18 · 10 周 · 4 个模块 · 1 周自己用
>
> **前置条件**: Phase 1 全部 ✅, 包括"自己用一周"。

---

## 这一阶段要达成什么

**18 周后, 我能在 iPhone 上做这件事**:

> 某天, APP 主动找我一次。
> 它说: "我从你最近的几条信号里看到一条可能值得下注, 要不要聊聊。"
> 我点开, 进入五轮追问对话, 它问我对这条信号的认识。
> 五轮过完, 进入四道门评估。
> 如果四门全过, 它生成一份承诺书, 我看完, 按下签字按钮, 触感反馈。
> 我现在有了一份属于自己的"承诺"。

**这是 财富密码 的灵魂阶段——"对话 → 评估 → 承诺"**。

如果说 Phase 1 是"水管", Phase 2 是"水里的仪式感"。每一个交互都是产品的核心价值。

## 关键设计决策(Phase 2 专属)

### 决策 1 · 五轮追问的对话形态

不是一个长 form, 不是 chatbot 风格的流式对话, 是**"AI 出题, 你做选择"的卡片式对话**。

- 每一轮一张卡, 像信件一样
- 选择题为主, 开放回答为辅
- 选择题的选项包括"干扰项"和"漏选检测"
- 答错 / 漏选 → 系统给"诊断", 不直接给答案

详见 `产品文档/02_第二层_模式识别与推演训练.md` 和 `技术文档/05_Mastra_Agents与Workflows_大纲.md`。

### 决策 2 · 四道门的"沉默"

四道门**任一失败 → 系统沉默**, 不降级提示"虽然不够强但可以参考"。
这是产品哲学第二条 · 沉默优于发声的硬实现。

未过的信号去四个池:
- 门 1 失败 → 观察池(继续监听更多信号)
- 门 2 失败 → 教训池(标记"看到时已晚")
- 门 3 失败 → 日历池(等窗口到了再激活)
- 门 4 失败 → 舍弃池(不是你的菜)

### 决策 3 · 承诺书的呈现

承诺书**不是一份建议**, 是一份**契约**。视觉上:
- 报刊感(双横线、罗马数字、Playfair Display 标题)
- 黑底白字直角签字按钮(报刊风, 不是 Cupertino 圆角)
- 必须包含"你 6 个月后会感谢自己的理由"段落

详见 `产品文档/04_第四层_承诺执行.md`。

### 决策 4 · PDF 归档

签字成功后, 系统**生成一份 PDF 归档**, 这是 财富密码 物质化承诺感的关键设计。

- 用 chromedp(Chromium headless) 渲染
- 模板用报刊风 HTML(可以复用原型 v4 的样式)
- PDF 存到对象存储, 用户随时能下载
- PDF 里有签字日期、四道门评估摘要、退出条件

## Phase 2 的"完成"定义

W18 自己用完后达成:

- ✅ 至少完成 1 次完整的"信号 → 五轮追问 → 四道门 → 承诺书 → 签字"流程
- ✅ 至少出现 3 次"四道门未过 → 沉默归档"(验证沉默机制工作)
- ✅ 签字过的承诺书有 PDF, 内容正确
- ✅ 签字过程符合 native_feel_skill 里所有约束(触感、防双击、报刊感)
- ✅ 整个对话流程, 没有任何 toast / loading / Material Ripple 出现
- ✅ 五轮追问每轮 < 60 秒答完, 不卡

**如果至少 1 次"真签字"没发生**(因为四道门确实没过), 那允许 W18 用模拟数据验证。
但**真签字才是产品验证**, 在 W26 前必须有一次真实签字。

## Phase 2 最大的风险

**LLM 输出不稳定**。

五轮追问的题目质量、四道门的判据严格度、承诺书的叙述风格, 全部依赖 LLM。
LLM 一不稳定, 产品就废了。

缓解:

1. **每个 Agent 写完后做 manual eval** — 用 10 个 fixture 信号跑一遍, 人工评分
2. **关键判据用规则引擎做兜底** — 四道门的部分判断(信号厚度计数、时间窗口)用代码, 不让 LLM 决定
3. **承诺书叙述的 prompt 写得极其详细** — 给 5 个 example, 限定字数, 列出 must-include
4. **写完 prompt 后做 property test** — 同一信号问 10 次, 输出形状应该一致

详见 M5-M7 的任务单。

## 进入下一阶段的"过关"问题

W18 自己用完后, 问自己:

1. 五轮追问让我感觉是"在被审问"还是"在和有判断的对手聊"? (后者过关)
2. 四道门把我的信号挡下来时, 我服气还是不服气? (服气过关)
3. 签字那一刻, 我有"这是一份契约"的感觉, 还是"我刚点了一个按钮"? (前者过关)
4. 承诺书 PDF 我会想保存下来吗? (会, 过关)

四问全过 → 进 Phase 3。
有问不过 → 在 W18 内修复对应模块。

## 给 AI Agent 的话(Phase 2 专用)

Phase 2 是**产品力的核心**, 不是工程力的核心。

容易犯的错:

- 把承诺书做成"投资建议", 用分析师语气, 而不是"给未来的自己看的"私人化语气
- 五轮追问的选项写得像"知识问答", 而不是"推演挑战"
- 签字按钮做成 CupertinoButton 蓝色圆角, 而不是黑底白字直角的报刊风
- 四道门未过时, 偷偷弹一个"虽然不够但可以考虑"的卡片(违反沉默)
- PDF 用通用模板, 没有报刊感

每一条都会让产品从"严肃契约"变回"又一个 AI 工具"。
