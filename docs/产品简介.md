# 财富密码（Cipher） · 产品简介

> 把模糊的高价值信号，转化为少数高确定性承诺的 AI 产品。

这是一份给产品方、投资方、第一次接触的协作者看的简介。读完它，你应该能回答三个问题：这个产品解决谁的什么问题、它长什么样、它凭什么能做到。

更深的内容：产品哲学见 [产品文档/06_产品哲学.md](产品文档/06_产品哲学.md)，技术全景见 [技术文档/README.md](技术文档/README.md)，当前架构图见 [architecture-iii.html](architecture-iii.html)。

---

## 一句话

**它替你守住那些"离信号最近、离仓位最远"的时刻。**

你常常是最早看懂一件事的人，却几乎从不在它上面下注——既包括市场里的下注，也包括项目里的承诺。财富密码 要修的，就是从"看懂"到"下注"中间那条全程失灵的链路。

---

## 它解决的痛点

> "1 月龙虾爆火那阵，群里清一色在讨论怎么抢 Mac Studio 512G，'内存带宽是瓶颈'这话翻来覆去说了不下十遍。机器抢到了，跑模型跑得飞起。直到最近回头一看内存股的走势——合着技术看得透透的，钱全花在消耗品上，相关的股一股没买。"

不是没识别出信号，而是从"信号"到"承诺"中间的链路全程失灵。这条链路上，大多数人缺的不是信息，而是**看见自己**的能力——看不见自己在哪一秒动过念头、又在哪一步退缩了。

---

## 核心信念

- 真正的 alpha 来自**极少数**高确定性机会，一年只出现 1–2 次。
- 识别它们的能力**不可教，但可以练**。
- 机会出现时，你必须同时有三样东西：识别它的前瞻、下注的胆量、持仓的耐心。

产品把这三件事**工程化**，而不是当成"性格问题"。

---

## 产品形态

不是 dashboard，是一个**对话式收件箱**。

它**默认安静**：一年只主动找你 1–2 次，每次都是"我从你最近的输入里发现一个可能值得下注的事，要不要展开聊聊"。点开后才进入深度交互，其余时候它在后台默默工作、默默归档。

> **产品的频率必须匹配 alpha 的频率。** 永远在响的产品，背叛了"alpha 来自少数机会"这条世界观。

## 五层架构

```
1 · 信号捕捉层    抓住一线观察, 不让它蒸发
       ↓
2 · 模式识别层    把现象翻译成资产、把目标翻译成里程碑(五轮追问训练)
       ↓
3 · 确定性评估层  投决会, 全票过会才提示, 任一否决就沉默
       ↓
4 · 承诺执行层    把判断压缩成一次签字, 把焦虑映射回承诺书
       ↓
5 · 复盘训练层    让你看见自己, 让蒸发的瞬间凝结成水
```

| 层 | 它做的事 | 关键设计 |
|---|---|---|
| **L1 信号捕捉** | 30 秒记下一条口语化观察 | 双通道录入；离线可写，重连即同步 |
| **L2 模式识别 / 推演训练** | 把现象翻译成资产，并用"五轮追问"练你的推演 | AI 一次只问一个问题，给选项而不给答案 |
| **L3 确定性评估** | 投决会，全票过会才提示，任一否决就沉默 | 判断永远在规则引擎里，不交给 LLM |
| **L4 承诺执行** | 全票过会 → 生成承诺书 → 一次签字 | 承诺书不是建议；签字之后其他全部预决定 |
| **L5 复盘训练** | 承诺到期，陪你回看完整时间线 | 一次复盘只产出一条改进方向 |

每一层都共享同一个底座：**你的个人知识图谱**——你说过什么、在乎什么、赢过 / 输过什么。

各层详述见 [产品文档/](产品文档/) 下的 01–05 五份文档。

## 两条主线：项目 与 投资

产品底层是同一件事——**把稀疏的、藏在日常表达里的高价值认知，组织成一个可执行的形状**——但它同时服务两个诉求：

- **想法 1 · 项目**：项目目标 → 关键里程碑 → 自动拆任务
- **想法 2 · 投资**：技术信号 → 投资机会 → 下注与持仓

落到产品里，就是**项目分组**：每条信号可以归属一个"项目"(一次投资主题，或一个工作目标)，项目带一段 `guidance`，引导后台推演往哪个方向想。同一套"捕捉 → 推演 → 承诺"的链路，既守市场里的下注，也守项目里的承诺。

---

## 注意力诊断：把"看见自己"做成高频前哨

复盘按承诺**到期**触发，一年也就几次。但"让自己可见"不该只发生在到期那天。

所以每完成一次五轮追问推演，系统会立刻在四个维度给这次思考打分，并写一句话画像加一条具体盲点：

| 维度 | 量的是 |
|---|---|
| **专注 focus** | 答题节奏稳不稳，有没有仓促秒选或分心拖延 |
| **深度 depth** | 有没有推到二阶 / 三阶，还是被表层诱导项带偏 |
| **广度 breadth** | 展开了几个独立视角，还是从头到尾一个角度 |
| **执行 execution** | 最后一步落不落地：动作 + 持仓时长 + 含退出条件的理由 |

盲点必须落到下一次的**具体动作**，而不是"多观察 / 再思考"。它和时间线回放是同一件事的两个尺度——**时间线让你看见一次完整下注里的自己，注意力诊断让你看见每一次思考里的自己**。

## 底层引擎(给技术读者)

产品的克制不是 UI 风格，是从架构里长出来的：

- **事件溯源是事实模型**：`events` 表 append-only，记录你和每件事接触的每一个瞬间——这正是 L5 能把"蒸发的瞬间"放给你看的物理前提。其他表都是可重建的物化视图。
- **LLM 准备数据，规则引擎做判断，LLM 解释结果**：投决会的判断永远在 Go 里跑；Mastra(基于 Claude)只负责"把模糊变结构化"和"把结构化变自然语言"。**判断动作不外包给 LLM**，这是产品的良心所在。
- **默认沉默**：绝大多数代码路径的终点是"归档"而不是"通知"。
- **异步靠 iii 编排引擎**：用户离开屏幕后才发生的推演走事件 outbox → iii 队列 → Mastra worker，至少一次投递、失败重试进 DLQ；用户在等的事则同步低延迟返回。

技术栈一览：Expo / React Native 客户端 · Go + Gin + Ent · PostgreSQL + pgvector · Mastra(Node)编排 Claude · iii 异步编排引擎。

## 五条反直觉的产品哲学

1. **沉默优于发声** — 大部分时候，这个产品不主动找你。
2. **频率匹配 alpha** — 一年 1–2 次主动提示，不是每天 5 条。
3. **减少决策，而不是增加信息** — 一次签字，其他全部预决定。
4. **承诺书不是建议** — 语言塑造行为。
5. **让用户看见自己，比让用户赚钱更重要** — 复盘与注意力诊断才是真正的主体。

| Phase | 周数 | 模块 | 完成时能做什么 |
|---|---|---|---|
