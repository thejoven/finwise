## § 0 · 一句话与它为什么重要

**一句话**:给每个信号 / 承诺的标的,以"你发现它的那一刻"为锚点,缓存并展示其后的日线走势与累计涨跌,点击标的进入它的专页,看见你所有碰过它的命题 + 这条价格路径。

**为什么重要——它补上产品缺的那一环**:
现在的产品帮你**想**(捕捉→精炼→四道门→承诺→复盘),却从不告诉你**想对了没有**。复盘环节问的是"你当时怎么想",却没有"后来怎么样"。标的追踪是这条闭环上缺失的**结果层 / 地面真值**:

- 让**复盘诚实**:你放下的、你押下的,事后价格给出客观回答。
- 给 **[12 推荐](12_主动信号推荐_开发文档.md)** 喂真值:画像才能从"像你过去"进化到"像你看得准"。
- 给用户**体感**:一条绿/红的曲线,比任何文案都更能校准判断力。

不做投资建议、不报实时 tick、不出买卖提示——只呈现"发现后,它怎么走了"。与"不替用户下单 / 移钱"的既定立场一致。

## § 3 · 三个硬问题(诚实面对)

### 硬问题一:标的归一(symbol resolution)

beneficiary agent 现在吐的是人话/混杂代码。要追价格,必须先归一到 **{规范代码, 交易所, 市场}**。

**方案**:

1. **升级 beneficiary**(及 analyst)产出**结构化**标的:`{name, symbol, exchange, market}` 而非裸 ticker;旧数据走一次性 resolver 回填。
2. **Go 侧 resolver + `assets` 注册表 + 别名表**:LLM 归一后,在 `assets` 里 upsert 规范记录;命中别名(`asset_aliases`)直接复用,省 LLM。
3. **人工兜底端点**:LLM 必然有归一不了的(同名、新股、非上市受益方)。提供手动指定 / 标记"不可追踪",**诚实留空好过追错**(呼应 [信号永不未分类](../../server/internal/module/signal) 的"宁可诚实兜底也不给错值")。

### 硬问题二:行情源(关键决策)

需要 A股/港股/美股的**日线 OHLC**(实时/分钟线非必须)。约束:产品中文向、部署在 **205 且在 GFW 后**。

| 选项 | 覆盖 | 反 GFW / 可达 | 代价 |
|---|---|---|---|
| Tushare Pro | A股强、港股可、美股弱 | 国内可达 ✓ | 需 token、有频控/积分 |
| AkShare | A/港/美都广 | 国内可达 ✓ | Python 库,需起 sidecar 进程 |
| 东方财富/新浪 非官方 | A/港/美(分端点) | 国内可达 ✓ | 免费但脆、无 SLA、易变 |
| Alpha Vantage / Finnhub / Polygon | 美股强、A股弱 | GFW 后可达性存疑 | 美股为主,A股不行 |

**建议**:做**行情源抽象**(interface + adapter,照 [LLM model 工厂](../../mastra/src/llm/model.ts) 与 [twtapi client](../../server/internal/infra/twtapi/client.go) 的可换源风格),把"选哪家"与"怎么用"解耦。默认源**待 205 实测可达性后定**(见 §8 决策一)。先用一家把 A股跑通,港美随抽象扩。

### 硬问题三:锚点("发现后"从哪天起)

"发现后"必须有明确、**冻结**的起点,否则收益数字可被事后挑拣,失去诚实性。

- **锚点 = 信号 `captured_at`**(你/系统登记它的时刻)为规范"你发现的时刻"。
- 对**转信号的推文**,底层 `tweets.tweet_created_at` 是"信号真正出现"的更早时刻——作为**次要标记**叠加展示("原文发布于…"),但收益锚点仍用 `captured_at`(那是**你**发现的时刻,不是别人发推的时刻)。
- 锚点在归一时**冻结**写入 `signal_assets.anchor_at`,此后不随重算变动。
- 收益 = `(最新收盘 − 锚点日收盘) / 锚点日收盘`。

## § 4 · 数据模型

派生/缓存数据,**不写 events**(同既有先例)。

**`assets`** —— 规范标的注册表

```sql
CREATE TABLE assets (
    id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    canonical     text NOT NULL,        -- 规范代码,如 "300750" / "00700" / "NVDA"
    exchange      text NOT NULL,        -- "SZSE" / "SSE" / "HKEX" / "NASDAQ" …
    market        text NOT NULL CHECK (market IN ('a','hk','us')),
    name          text NOT NULL,
    provider_symbol text,               -- 行情源各自的代码(如 "300750.SZ")
    type          text NOT NULL DEFAULT 'equity',
    status        text NOT NULL DEFAULT 'active',  -- active|delisted|untrackable
    created_at    timestamptz NOT NULL DEFAULT now(),
    UNIQUE (market, canonical)
);
```

**`asset_aliases`** —— LLM/用户给的各种叫法 → asset_id(省重复归一)

```sql
CREATE TABLE asset_aliases (
    alias_lower text PRIMARY KEY,       -- 归一化后的别名("宁德时代" / "ningde" / "catl")
    asset_id    uuid NOT NULL REFERENCES assets(id)
);
```

**`asset_prices`** —— 日线缓存

```sql
CREATE TABLE asset_prices (
    asset_id  uuid NOT NULL REFERENCES assets(id),
    date      date NOT NULL,
    open      numeric, high numeric, low numeric, close numeric NOT NULL,
    volume    bigint,
    source    text NOT NULL,
    PRIMARY KEY (asset_id, date)
);
```

**`signal_assets`** —— 信号 ↔ 标的链接 + 冻结锚点(**取代**只靠 JSON 查)

```sql
CREATE TABLE signal_assets (
    signal_id  uuid NOT NULL REFERENCES signals(id),
    asset_id   uuid NOT NULL REFERENCES assets(id),
    role       text NOT NULL DEFAULT 'beneficiary',  -- beneficiary | primary | mentioned
    anchor_at  timestamptz NOT NULL,                  -- 冻结 = 解析时的 signal.captured_at
    rationale  text,
    PRIMARY KEY (signal_id, asset_id)
);
```

> **为什么用链接表而非继续塞 JSON**:它一表服务三个查询——
> ① 本文的"某信号/承诺的标的走势";
> ② [12 推荐](12_主动信号推荐_开发文档.md) 的"关于标的 X 的新情报";
> ③ 反查"标的 X 涨了 → 我哪些命题碰过它"(标的专页的核心)。
> JSON 做不到第②③。承诺侧同理可加 `commitment_assets`,或承诺直接复用其 evaluation→refinement→signal 链路反查。

## § 5 · 各层改动

**Go**:

- `internal/infra/marketdata/` —— 行情源抽象 + 首个 adapter(interface、`IsConfigured()`、typed errors、env 配置,照 [twtapi](../../server/internal/infra/twtapi)/[mastra client](../../server/internal/infra/mastra/client.go))。
- `internal/module/asset/`:
  - `resolver.go` —— 归一(别名命中优先,未命中调 Mastra/规则)→ upsert `assets`/`asset_aliases`/`signal_assets`(冻结锚点)。
  - `poller.go` —— 价格回填 + 日更 worker(照 [subscription poller](../../server/internal/module/subscription/poller.go):新标的从 `anchor_at` 回填;每日 tick 给所有被追踪标的补最新 bar;`SKIP LOCKED` 取批;配额/退避处理频控)。
  - `handler.go` —— 读端点:
    - `GET /v1/assets/:id/prices?from=&to=` —— 原始日线
    - `GET /v1/signals/:id/track` —— 解析该信号标的,返回 `[{asset, anchor_at, anchor_close, latest_close, pct_change, series[]}]`
    - `GET /v1/commitments/:id/track` —— 承诺标的,锚点可叠加签字日(最高价值入口)
    - `GET /v1/assets/:id/theses` —— 反查"我碰过它的所有命题"(标的专页)
    - `POST /v1/assets/resolve`(内部/人工兜底)—— 手动指定 / 标记不可追踪

**Mastra**:

- 升级 [beneficiary.ts](../../mastra/src/agents/beneficiary.ts) 产出结构化 `{name, symbol, exchange, market}`(向后兼容旧裸 ticker)。
- 可选 `symbol-resolve` 端点供 resolver 调用做歧义归一(同六端点同形)。

**Mobile**(`src/features/track/`,图表复用 [attention 的图表库](../../mobile/src/features/attention/AttentionScreen.tsx)):

- **可复用价格组件**:折线(收盘),锚点处一条"发现"竖线标记,顶部醒目累计涨跌(涨跌色克制、遵循 [深色模式 token](../../mobile/src/shared)),承诺页叠加"签字日"标记。
- 落点:
  - **信号详情** `/signal/[id]` —— 每个标的一条 sparkline + "发现至今 %"。
  - **降噪行** —— 受益标的旁一枚微型 sparkline + 涨跌角标(一眼看出"这条读对没")。
  - **承诺/持仓** `/commitment/[id]` —— 锚定签字日的完整曲线,"你这笔承诺的表现"(情绪兑现点)。
  - **归档/复盘** —— 被否信号标的的价格路径:"你当时放下的,后来怎么样了"——校准四道门是否否对了。
  - **标的专页**(点击任意标的):完整曲线 + `GET /assets/:id/theses` 列出你碰过它的每条信号/承诺。**这正是用户原话"点击后查看对应发现后的股价变动"。**
- API 客户端 `core/api/track.ts`(照 [gate.ts](../../mobile/src/core/api/gate.ts))。

## § 6 · 分期 —— "目前阶段先做哪一刀"

| 期 | 名称 | 范围 | 为什么这个顺序 | DoD |
|---|---|---|---|---|
| **P0** | 标的归一 | `assets`/`aliases`/`signal_assets` + resolver;升级 beneficiary 结构化;回填存量 | 一切的前提;无价格也能先验证"归一对不对" | 存量 `related_assets` 归一命中率人工可接受;兜底端点能标不可追踪 |
| **P1** | 行情底座 | marketdata 抽象 + 选定源 + `asset_prices` + 回填/日更 poller + `GET /assets/:id/prices` | 205 实测可达性后定源,先把 A股日线跑通 | 某 A股标的从锚点回填到今日、日更不断 |
| **P2** | 承诺表现(最高价值) | `/commitments/:id/track` + 承诺/持仓页完整曲线(锚签字日) | "我这笔押对没"是情绪兑现点,直接 | 某签字承诺页出现锚定曲线 + 累计涨跌 |
| **P3** | 信号/降噪 sparkline | 信号详情 + 降噪行的"发现至今 %"微图 | 把结果带到日常浏览,纯增量 | 降噪行一眼见涨跌 |
| **P4** | 标的专页 + 归档联动 + 喂给推荐 | 点击标的→专页(全命题反查);归档信号价格路径;`signal_assets` 收益暴露给 [12](12_主动信号推荐_开发文档.md) P4 | 完成用户原话的"点击查看";闭合到推荐真值 | 标的专页通;归档"放下后涨了吗";12 能读到收益 |

**当前阶段建议**:**P0 → P1 → P2**。P0/P1 是地基(没有归一和行情,后面全空中楼阁);P2 是用户最先想看见的"我押对没"。P4 与 [12](12_主动信号推荐_开发文档.md) 排程对齐。

## § 7 · 数据诚实、口径、失败模式

- **诚实兜底**:归一不了 → `status='untrackable'`,UI 显示"无法追踪此标的",**绝不**用模糊匹配凑一个可能错的代码(错的价格比没价格更有害)。
- **锚点冻结**:`anchor_at` 一经写入不随重算变动,杜绝事后挑拣起点。
- **数据出处**:UI 标注行情来源 + 数据 as-of 日期;停牌/退市显式处理(`assets.status='delisted'`,曲线断点)。
- **不是投资建议**:无实时 tick、无买卖提示、不替用户下单/移钱(既定立场);仅"发现后它怎么走"。
- **失败模式**:行情源不可用 → 已缓存照常展示 + 标"数据截至 X";新标的回填失败 → 重试 + 标"暂无数据",不阻塞信号主流程。
- **成本/频控**:全局标的去重(多个信号同一标的只缓存一份,同订阅 `tweets` 全局共享精神);poller 退避处理源频控。

## § 8 · 开放决策(执行前需拍板)

1. **默认行情源** —— 需先在 205 实测各源可达性 + 覆盖,再定默认 adapter(§3 硬问题二)。建议先 A股一家跑通。
2. **市场范围** —— 首发只 A股,还是 A+港+美?抽象层支持全部,但首个 adapter 覆盖面影响 beneficiary 归一与回填工作量。**这是 13 的头号产品决策。**
3. **承诺标的链接** —— 新建 `commitment_assets` 链接表,还是复用 evaluation→refinement→signal→`signal_assets` 反查?建议后者(少一张表,链路已存在)。
4. **日线 vs. 分钟线** —— "发现后变动"日线足够;分钟线留作后续可选,不进首发。
