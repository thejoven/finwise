## § 0 · 一句话与它为什么重要

**一句话**:第一次进入时,用 6 道「答完即揭示一个功能」的问题完成个性化与教学,合成一张"为你建立的 AlphaX",再进三档付费墙;邀请码用户走免费内测、无码用户付费,购买后落到财知。

**为什么重要**:

- 现在 splash 之后**直接进 tab**,新用户对着一个信息密度极高的财经工具一脸茫然——没有任何"这东西能为我做什么"的铺垫,留存与付费都无从谈起。
- 「一问一功能」把**教学**(每题揭示一个功能)、**个性化**(答案塑造首屏内容与成长模式种子)、**付费铺垫**(答完已投入 6 次选择、看见全貌)三件事叠在同一段流程里——这是消费级工具付费墙转化最高的结构。
- 付费分层把最贵的能力(成长模式 = 持续记忆 + 跨信号 RAG,真实算力成本)放进高档,让定价与成本对齐,也给"按周尝鲜 / 按月用全 / 终身买断"三类心智各自的入口。

**与产品既定立场一致**:不做 FOMO、不做投资建议、报刊式克制。付费墙文案中性直说能力,不挑衅、不制造焦虑(呼应 [12](12_主动信号推荐_开发文档.md) 的"宁可不推"红线与全局克制哲学)。

## § 3 · 硬问题(诚实面对)

### 硬一:"三档不同内容" 撞二元 `isPro`

现在客户端只投影一个布尔 `isPro`([store.ts](../../mobile/src/core/billing/store.ts) `project()`),服务端只认一个 `"pro"`。三档不同内容必须有"档位"概念。

**方案——其实是 2 个内容档 × 3 个 SKU**:

| SKU | 价格 ⚠️占位 | 内容档 | 资讯订阅 | 成长模式 | RevenueCat entitlement |
|---|---|---|---|---|---|
| 周卡 · 体验 | ¥18 / 周 | Entry | 限量(如 3 源) | ✗ | `entry` |
| 月卡 · Pro | ¥68 / 月 | Full | 不限 | ✓ | `pro` |
| 终身 · 创始 | ¥698 一次 | Full | 不限 | ✓ + 未来功能 | `pro` |

- **真正的内容分界线只有一条**:Entry vs Full。月卡与终身**内容完全相同**,只差租(订阅续费)vs 买(一次性非消耗)。这比"三档各不同"诚实——用户一眼算得清账,也躲掉"月卡比终身贵却更少"的悖论。
- **两个 entitlement**:`entry`、`pro`。周卡产品授予 `entry`;月卡 + 终身产品授予 `pro`。**无需迁移**(030 表已支持多行)。
- **客户端投影出 tier**:`project()` 改为 `hasPro ? 'full' : hasEntry ? 'entry' : 'free'`,导出 `useTier()`;`useIsPro()` 保留为 `tier === 'full'` 的别名(向后兼容现有调用)。
- **服务端认两档**:`EntitlementID` 单常量 → `entitlementIDs = {"entry","pro"}`,webhook 的 `containsEntitlement` 与 `GetEntitlement` 按档查;`GET /v1/billing/entitlement` 返回结构加 `tier` 字段。
- **价格不进代码**:由 RevenueCat offering 动态下发,改价/改文案不发版(现有 [ui.tsx](../../mobile/src/core/billing/ui.tsx) 托管 Paywall 即此机制)。
- **必修**:把 entitlement identifier 定为规范的 `entry` / `pro`(小写,与 RevenueCat 控制台一致),修掉 config.ts 的 `"AplhaX Pro"` typo 默认值。

### 硬二:「成长模式」是什么、加在哪

**定义**:成长模式 = AI 记住用户的投资偏好、关注方向,并在推演时跨历史信号召回线索,越用越懂你。它**不是新功能**,是把三块已有地基收成 Full 专属:

1. **per-user 记忆**([agent-memory.ts](../../mastra/src/memory/agent-memory.ts)):resourceId=user_id 的工作档案(持仓/lens 倾向/历史决策/红线),已在 main。
2. **Tier1 RAG**([analyst](../../mastra/src/skills/analyst/index.ts) `recallSimilar` top-6 + [socratic](../../mastra/src/agents/socratic.ts) 接 `prior_inference` 再追一层):跨信号线索汇总,**代码已完成、未部署**。
3. **画像 `user_alpha_profile`**(迁移 024 + [recommend builder](../../server/internal/module/recommend)):tag/category/conviction/active_theses 等派生维度,已在 main。

**门禁注入点(两处,服务端权威)**:

- **成长模式开关**:Go 在触发推演前用 `billing.GetEntitlement` 解析 tier,把 `growth_mode: bool` 随请求经 [mastra client](../../server/internal/infra/mastra/client.go) 透传给 [signal-inference 工作流](../../mastra/src/workflows/signal-inference.ts)。`growth_mode=false` 时:analyst 跳过 `recallSimilar`(无跨信号 RAG)、socratic 不注入 `prior_inference`、记忆走临时(不写 resourceId 长期档案)。Full 用户才拿到完整闭环。
- **诚实点**:Tier1 RAG 现在没部署 → 直接**带门上线**,不存在"从免费用户手里回收"的问题;而记忆虽已在 main,但当前用户全是受邀内测(= 免费 Full,见硬三),门禁对他们无感。

> 为什么门放服务端而非客户端:能力值钱、客户端可篡改。客户端的 `useTier()` 只用于 UI 呈现(显示锁/升级),真正"跑不跑 RAG"由 Go 解析 entitlement 决定。

### 硬三:受邀制 × 自助付费墙 并存

现在注册受邀制([invite 模块](../../server/internal/module/invite),码必填)。加自助付费墙后,产品拍板:**邀请 = 免费内测 / 付费 = Pro,两条并行**。

**方案——单漏斗 + 邀请码作 escape hatch**:

- 所有人走**同一套** onboarding 6 题(教学+个性化对谁都有用)。
- 到付费墙时:
  - 无码用户 → 选档付费。
  - 有邀请码 → 付费墙底部「有邀请码?」→ 输码 → 解锁**免费 Full(内测)**,跳过付费。
- **内测 = comp entitlement**,复用 `user_entitlements`:`account.Register` 成功兑换邀请码后,后端给该用户 upsert 一行 `(user_id, 'pro', is_active=true, store='comp', expires_at=NULL, last_event_id='comp:'+user_id)`。**所有门禁统一读 `user_entitlements`**——付费(RevenueCat webhook)与内测(invite 授予)同一条门禁路径,不分叉。
  - 边界:纯内测用户没有 RevenueCat 事件,comp 行永不被 EXPIRATION 覆盖(webhook 只对该用户真实购买触发);若内测用户日后真购买,webhook upsert 同行覆盖为真实到期——平滑转正。
- **登录位置**:问完题 → 付费墙 → (购买或兑换成功后)→ 注册/登录 → 进财知。账号在购买前后都可建(RevenueCat 匿名 appUserID,登录后 `identify` 迁移,[store.ts](../../mobile/src/core/billing/store.ts) 已支持)——**先价值后承诺**,转化更高。
- **free tier(无码且未付费)能看什么** ⚠️待定(§10):建议给一层"薄尝鲜"(浏览信箱 + 有限标的,AI 全锁),既满足 App Store"未付费也能见价值",又把付费墙立在 AI 能力前。

## § 4 · Onboarding 流程(一问一功能)

### § 4.1 · 整体序列

```
启动 SplashCover(已有,品牌动画)
  → Q1…Q6  每题:问题 + 选项 → 答完弹该功能微卡片 → 下一题
  → 合成画像「正在为你建立 AlphaX…」(把答案拼成首屏预览)
  → 付费墙(三档 + 底部「有邀请码?」)
  → 注册 / 登录
  → 财知 tab
```

- 落地为 expo-router 的一个路由组 `app/(onboarding)/`(welcome / q/[step] / building / paywall),首次完成后写本地标志位 `alphax.onboarded.v1`(AsyncStorage,同 [既有 store hydrate 机制](../../mobile/app/_layout.tsx)),冷启动时 `_layout` 据此 + auth/tier 决定首屏(已 onboarded + 已登录 → tabs;否则 → onboarding)。
- 每屏交互克制:一问一屏,大字问题(报刊衬线),2–4 选项,可单选/多选;答完一张轻量功能微卡片(一句话 + 一枚图标),不堆动画。
- 精简版可砍到 5 题(去掉 Q6 统计),A/B 由埋点定。

### § 4.2 · 6 道题映射

| # | 问题(克制语感) | 类型 | 揭示功能 | 答案落点 |
|---|---|---|---|---|
| 1 | 你的钱,主要在看哪些市场? | 多选:A股/港股/美股/加密/还在观望 | 标的追踪([13](13_标的追踪_开发文档.md),标的 tab) | `markets[]` → 首屏标的 + 画像 category 初值 |
| 2 | 你的灵感,多数来自哪里? | 单选:X 大V/研报/群消息/自己盯盘 | 信息源订阅 + 资讯工作台试滑([10](10_推文订阅_开发文档.md) / [15](15_订阅卡片滑动交互_开发文档.md),订阅 tab) | `sources` → 订阅推荐;揭示=交互式试滑(§4.4) |
| 3 | 看到一个像机会的消息,你通常会? | 单选:立刻上车/想想又算了/事后才反应 | 过会 · 苏格拉底追问([refinement](../../server/internal/module/refinement)) | `decision_style` → 画像 conviction 参考 |
| 4 | 最想让 AI 替你做掉哪件事? | 多选:过滤噪声/推演因果/盯兑现/复盘错误 | 降噪 + AI 推演 | `ai_wants[]` → 成长模式侧重 |
| 5 | 你记录过自己当时为什么买/不买吗? | 单选:从没/偶尔/一直有 | 承诺 + 复盘([commitment](../../server/internal/module/commitment)) | `journaling` → 复盘引导强度 |
| 6 | 你知道最近注意力都花在哪了吗? | 单选:知道/大概/没概念 | 统计 · 注意力分析([attention](../../server/internal/module/attention)) | `attention_awareness` |

- **答案双重用途**:① 即时个性化首屏(标的/订阅/分类预选);② **成长模式的第一颗种子**——落 `user_onboarding` 后由 recommend builder 混入 `user_alpha_profile` 初值(§6),Entry 用户拿到一次性个性化,Full 用户在此之上持续记忆。题目天然是付费功能的预告片。

### § 4.3 · 合成画像屏

把答案聚成一句"已为你接入":如「A股 + 美股追踪线 · 3 个信息源建议 · 你偏向"想想又算了"——过会会替你把犹豫拆开」。这是付费墙的临门一脚:让用户看见"系统已经懂我了",再请他付费解锁让它**继续**懂下去(成长模式)。文案中性陈述,不夸张承诺收益。

### § 4.4 · 资讯工作台的引导介绍(交互式试滑)

订阅 tab 的**资讯工作台**(原"创新阅读模式")= [15 卡片分拣台](15_订阅卡片滑动交互_开发文档.md)(一次一张:左滑已读 / 右滑转信号 / 下滑不感兴趣)。它手势驱动、**静态卡教不会**——引导要让用户**亲手滑一次**,建肌肉记忆 + 制造体感。

- **位置**:作 Q2(信息源订阅)的揭示,把静态功能微卡片换成一张**可滑的样例卡**。
- **教法(克制,不强迫三向)**:引导**右滑一次**(`转成信号 → 收进信箱`,最具差异化的"魔法动作"),完成即一张轻 toast;另两向(`← 已读`、`↓ 不感兴趣`)静态图例标注,想试可继续、「跳过」直接走。
- **零后端,复用现成组件**:直接用 [SwipeDeck](../../mobile/src/features/subscriptions/SwipeDeck.tsx) + [SignalCard](../../mobile/src/features/subscriptions/SignalCard.tsx),喂 3 条**本地样例** `TweetItem`,动作传 **no-op**(不落库 / 不建信号)、禁用点击进详情、叠 coach 浮层。需给 SwipeDeck 加一个可选 `demo` 模式(样例数据 / 教学浮层 / 动作空转)。
- **标记即训练(资讯工作台的内核)**:工作台里每次标记(右滑转信号=正 / 下滑不感兴趣=负 / 左滑已读)都是推荐流的训练信号 —— 标得越多,[12 主动信号推荐](12_主动信号推荐_开发文档.md) 的 `user_alpha_profile` / [15](15_订阅卡片滑动交互_开发文档.md) 的负反馈越准,推荐流越懂你(成长模式的日常入口)。引导在试滑成功后点出这句:"你标记的每一条,都会让推荐流越来越懂你"。
- **文案**:中性直说动作("右滑,转成信号"),不挑衅、不教程腔(YC PM 口径)。
- **双保险**:首次真正进订阅 tab 再浮一次 coach marks(可一键关),承接引导里学到的手势。

## § 5 · 分层模型与门禁(落到代码)

### § 5.1 · RevenueCat 配置(控制台,非代码)

- 一个 Offering `default`,3 个 package:
  - `$rc_weekly` → product `com.alphax.app.entry.weekly` → entitlement **`entry`**
  - `$rc_monthly` → product `com.alphax.app.pro.monthly` → entitlement **`pro`**
  - `$rc_lifetime` → product `com.alphax.app.pro.lifetime`(非消耗)→ entitlement **`pro`**
- 价格、试用、文案在控制台配,改动不发版。

### § 5.2 · Paywall 形态 ⚠️决策(§10)

两条路:

- **路 A · RevenueCat 托管 Paywall**([ui.tsx](../../mobile/src/core/billing/ui.tsx) `presentPaywall` 已封好):零自绘、远端可改,**最快**;但样式受模板限,报刊感弱。
- **路 B · 自绘整屏 Paywall**:用已导出的 `getCurrentOffering` / `purchase` / `restore`([purchases.ts](../../mobile/src/core/billing/index.ts))自己渲染三档卡片,完全贴 AlphaX 报刊视觉。

**建议**:鉴于品牌识别极强且 onboarding 末屏要无缝衔接,倾向**路 B 自绘**(动作 API 已备齐,成本主要在 UI);若要抢上线,先路 A、后替路 B。

### § 5.3 · 两道门

| 门 | 规则 | 注入点 | 失败行为 |
|---|---|---|---|

## § 6 · 数据模型

派生/偏好数据,**不写 events**(同 `user_alpha_profile` / `user_entitlements` 先例)。

**`user_onboarding`** —— onboarding 答案(新表,迁移 **031**)

```sql
CREATE TABLE user_onboarding (
    user_id      uuid PRIMARY KEY REFERENCES users(id) ON DELETE CASCADE,
    answers      jsonb NOT NULL DEFAULT '{}',  -- {markets:[],sources,decision_style,ai_wants:[],journaling,attention_awareness}
    completed_at timestamptz,                  -- NULL = 中途退出, 下次续问
    created_at   timestamptz NOT NULL DEFAULT now()
);
```

**`user_entitlements`** —— **复用,无迁移**(030 已支持多档):新增用法是同一 user 可有 `entry` 或 `pro` 行;内测用户写 `(user_id,'pro',store='comp',expires_at=NULL)`。

**`user_alpha_profile`** —— **复用**([12](12_主动信号推荐_开发文档.md) 迁移 024):builder 增一路"冷启动种子",把 `user_onboarding.answers` 的 markets→`category_affinity`、ai_wants→侧重、journaling→`conviction_shape` 参考混入初值(`sample_size` 不足时种子占比高,行为轨迹积累后自然稀释)。

**`invite_codes`** —— **复用**([invite 模块](../../server/internal/module/invite)):兑换成功触发 comp entitlement 授予,无 schema 变更。

## § 9 · 数据诚实、口径、失败模式

- **不做 FOMO / 不做投资建议**:付费墙与受限提示中性陈述能力,不承诺收益、不制造焦虑(承袭 [12](12_主动信号推荐_开发文档.md) 红线 + 全局克制)。
- **门禁单一真相**:能力门一律读服务端 `user_entitlements`;客户端 `useTier()` 只管 UI 呈现,不作授权依据(防篡改)。
- **成长模式降级是静默**:Entry/free 推演照常出结果,只是无记忆/无跨信号 RAG;不报错、不半路弹窗,UI 仅以"成长模式(Pro)"标注差异。
- **comp 与 webhook 不打架**:comp 行 `expires_at=NULL` 永有效;webhook 只对真实购买事件触发,纯内测用户不受影响;内测转付费时同行覆盖。
- **购买失败 / 网络**:RevenueCat 动作失败 → 留在付费墙 + 可重试 + "恢复购买"(防重复扣费);billing 未配 key 时全链路降级"未订阅"([config.ts](../../mobile/src/core/billing/config.ts) 已如此),onboarding 仍可走完(付费墙显降级态)。
- **onboarding 可中断**:`completed_at=NULL` 记进度,下次续问;答案缺失不阻塞进入(个性化退化为默认)。
- **App Store 合规**:价格/条款/恢复购买/隐私在付费墙明示;免费可见层(§10)确保"未付费也能见价值"。

## § 10 · 开放决策(执行前需拍板)

1. **价格数字** ⚠️:¥18/周、¥68/月、¥698/终身为占位,待产品定(RevenueCat 控制台配,不影响代码)。是否给周卡/月卡引入试用期?
