# 09 · 开发路线图与里程碑 · 大纲

> 作者视角:**Product Manager**
> 这份文档回答:先做什么、后做什么、每个 Phase 的验收标准、风险在哪里。

---

## 这份文档要回答的核心问题

1. 三个 Phase 各自的目标和边界?
2. 每个 Phase 的"上线"标准是什么?
3. 单人独立开发,每个 Phase 大概多久?
4. 哪些事我能边做边学,哪些必须前置准备好?
5. 风险在哪里,什么信号说明该停下来重新评估?

---

## 章节结构

### § 1. 三个 Phase 的设计哲学

每个 Phase 验证一件核心事:

- **Phase 1 · 验证沉默** — 一个低频、克制、不打扰的产品,自己用得下去吗?
- **Phase 2 · 验证承诺** — 当投决会全票过会时,我自己会按下签字键吗?
- **Phase 3 · 验证陪伴 + 复盘** — 持仓期间,产品能不能拦住我割肉?复盘能不能让我看见自己?

**Phase 不是功能集合,是世界观验证步骤**。这一点必须在开始前就清楚。

### § 2. Phase 1 · 验证沉默 (6 周)

**目标**:能自己每天用 30 秒录一条信号,后台 AI 默默推演完归档。**APP 一周不主动找我一次,但我每天打开都能看到自己最近的输入正在被理解。**

**范围**:

- ✓ RN B1 录入页 + A1 收件箱(只读)
- ✓ Go signal + inference + 极简 knowledge_graph 模块
- ✓ Mastra Analyst Agent (tagging + background_inference 模式)
- ✓ events 表 + 物化视图基础
- ✓ iii 异步推演链路(events outbox → iii → Mastra)
- ✓ Phase 1 部署形态(VPS + Docker Compose)
- ✗ 投决会评估 (Phase 2)
- ✗ 五轮追问 (Phase 2)
- ✗ 承诺书 (Phase 2)
- ✗ E4 焦虑陪伴 (Phase 3)
- ✗ 复盘 (Phase 3)

**验收标准**:

- [ ] 录入一条信号,30 秒内 UI 显示"已归档"
- [ ] 5 分钟内后台推演完成,可在详情页看到"AI 推演:谁变富、谁变穷"
- [ ] 累积 30 条信号后,APP 没有主动推送过一次
- [ ] 离线录入的信号在重连后自动同步
- [ ] 每天打开 APP 一次,平均使用时长 < 1 分钟

**周分解**(6 周):

- W1:Expo 项目初始化 + 字体加载 + 基础 theme + Go 数据库 schema
- W2:Go signal 模块 + events 表 + iii 集成(event_outbox → iii engine)
- W3:Mastra Analyst Agent + Workflow + Go-Mastra 双向接口
- W4:RN B1 录入页 + WatermelonDB + Sync Queue 端到端跑通
- W5:RN A1 收件箱 + 基础 Masthead + 列表呈现
- W6:部署 + 自己用一周

**最大风险**:

- LLM 推演结果不稳定 → 用 property test 早期发现
- RN + WatermelonDB 离线同步状态机复杂 → 用 TanStack Query mutation 简化
- Expo + Mastra 字体和 SSE 集成可能踩坑 → W1 优先打通最小链路

### § 3. Phase 2 · 验证承诺 (8-10 周)

**目标**:能从一组信号走到一张签好字的承诺书,自己愿意按那个签字键。

**范围**:

- ✓ Mastra Socratic Interviewer (五轮追问)
- ✓ Go gate 模块完整实现(投决会规则引擎)
- ✓ Mastra Narrator Agent + Commitment Narration Workflow
- ✓ Go commitment 模块 + PDF 渲染
- ✓ RN 五轮追问对话页 + 承诺书 + 签字
- ✓ 选择题训练(L2)基础版

**验收标准**:

- [ ] 历史信号回放,系统能识别"内存稀缺"这种 thesis cluster
- [ ] 投决会全票过会的场景出现至少 1 次(可能需要手动喂数据)
- [ ] 承诺书叙述能感动到自己(自评 + AI 评估)
- [ ] 我自己愿意按下签字键

**周分解**:

- W1-2:Mastra Socratic + RN 五轮追问 UI(SSE 流式)
- W3-4:Gate 规则引擎 + fixture 测试
- W5-6:Narrator Agent + PDF 渲染
- W7-8:RN 承诺书 + 签字流程(触感 + 防双击 + 报刊感)
- W9-10:Buffer + 自己用两周

**最大风险**:

- 投决会评估在真实数据上长期不触发 → Phase 1 录入数据可能 thesis 太分散
- 承诺书文案写不出"那种感觉" → Narrator Agent prompt 反复迭代

### § 4. Phase 3 · 验证陪伴与复盘 (8 周)

**目标**:持仓期间产品真的能拦住自己割肉。到期能看见自己的成长。

**范围**:

- ✓ Editor Agent (焦虑陪伴) + E4 卡片触发逻辑
- ✓ Go anxiety + exit_monitor 模块
- ✓ Diagnostician Agent (复盘四问)
- ✓ RN E4 + F2 时间轴
- ✓ 退出条件每日巡检
- ✓ 选择题训练完整版

**验收标准**:

- [ ] 持仓 30 天内,出现至少一次"想反悔"的真实时刻,E4 卡片被触发
- [ ] 看完 E4 卡片后,我没有割肉
- [ ] 持仓到期能完整跑一遍复盘对话
- [ ] F2 时间轴能让我看到自己 6 个月的接触瞬间
- [ ] 系统给出的"下次训练重点"我认同

**周分解**:

- W1-2:E4 卡片 + Editor Agent
- W3-4:退出条件巡检 + 行为指纹 + RN 端打开次数追踪
- W5-6:复盘 Diagnostician + F2 时间轴(RN 自绘纵向)
- W7:选择题训练补完
- W8:自己用 + 复盘

### § 5. Phase 0 · 准备期(可选,2 周)

如果想在 Phase 1 前做更多准备:

- 把产品文档六份再深读一遍
- 跑通 hello world(Expo + Go + Mastra 三端通信)
- 把 events 表 schema 在纸上画一遍
- 调通 OpenAI / Anthropic API key
- 装好 Xcode + iOS Simulator + Expo CLI + EAS CLI

我的建议:**Phase 0 不超过 1 周**,边做边学比前置准备效率高。

### § 6. 不在路线图里的事

明确不做的事,避免范围蔓延:

- 多用户支持(Phase 3 之后)
- 推送通知(永远不做,产品哲学)
- 实时行情接入(Phase 3 之后,且只是退出条件巡检的数据源)
- 社交分享(永远不做)
- Web 端(Phase 4+,如果需要)

### § 7. 风险与早期预警

每个 Phase 列出风险信号,出现就停下来评估:

**Phase 1 红灯**:
- 录入信号变成"任务感",我自己都不想用
- LLM 推演结果质量差,看了想关 APP

**Phase 2 红灯**:
- 投决会长期不触发(说明判据太苛刻或者数据不够)
- 承诺书文案我自己都说服不了自己

**Phase 3 红灯**:
- E4 卡片出现时,我反而觉得被打扰
- 复盘对话变成"AI 在评分我"

红灯出现:**停下来,不要堆功能**。回到产品哲学,看哪里偏了。

### § 8. 度量这个产品的健康度

不用 DAU、留存这种指标。用:

- **沉默率** — 每周 APP 主动推送 0 次的天数 / 7
- **录入到归档延迟** — 30 秒内完成的比例
- **过会率** — 不追求高,符合"高确定性稀少"
- **持仓违约率** — 用户在退出条件未触发时主动平仓的次数

目标:沉默率 > 95%,过会率 1-2 次/年,违约率 0。

### § 9. 单人开发的节奏建议

- 一次只做一件事。文档说克制,代码也要。
- 每个 Phase 结束后,**强制休息 2 周**,自己用产品,不写代码。
- 用 AI 协助写代码可以,但 prompt 设计、规则演化、复盘四问这些核心逻辑必须自己想清楚。

---

## 关键决策预告

1. **Phase 是世界观验证,不是功能集合** — 每个 Phase 验收标准都是体验,不是功能
2. **每个 Phase 结束强制休息 2 周** — 自己用,才知道下一步该做什么
3. **不在路线图里的事就是不做** — 防止范围蔓延
4. **红灯就停** — 加功能不能掩盖产品哲学跑偏
