# ADR 0006: "投决会 / 过会" 作为第三层评估的总称（承接 0005）

- 状态: Accepted
- 日期: 2026-06-08
- 相关: [ADR 0005 (Named analysts over 四道门)](0005-analysts-over-gates.md) · [第三层 · 确定性评估](../产品文档/03_第三层_确定性评估.md)

## Context

ADR 0005 把四个判据从"门 1 / 门 2 / 门 3 / 门 4"改名为**四位分析师**（佐证 / 共识 / 时机 / 能力圈），解决了"每道门叫什么"。但**这一层、这一关的总称仍叫"四道门"**——一个闸门 / 关卡的物理隐喻：冷、被动，而且和"分析师"这个人格化设定打架（一边是有判断、有口吻的分析师，一边却被装进四道铁门里）。残留遍布代码注释、产品文档、UI 按钮（"进入四道门" / "正在交给四道门…"）。

## Decision

### 1. 总称改为「投决会」（投资决策委员会 / 投委会）

金融业里"**一组独立专家逐一审议、全票通过才放行、任一否决即毙掉**"这个动作的现成词，就是**投资决策委员会（Investment Committee, IC）**。它是"四位分析师"在逻辑上的**必然总称**：

- 四位分析师 = 投委会的**四个席位 / 委员**（个体 → 集体，同一隐喻的两层，零成本复用）
- 它强化产品的核心叙事——**把机构级的决策纪律装给个人投资者**："给你自己配一个投委会，冲动信号得过会才放行。"

### 2. 词汇表（单一事实源）

| 概念 | 旧（老派） | 新（金融术语） |
|---|---|---|
| 这一层 / 这一关（名词总称） | 四道门 | **投决会**（投委会） |
| 四个判据角色 | 门 1..4 | **四位分析师**（= 四个席位，沿用 0005） |
| 进入评估（动词） | 进入四道门 / 交给四道门 | **上投决会 / 上会** |
| 全过（结果） | 四门通过 / 全过 | **全票过会 / 过会** |
| 任一 fail（结果） | 某道门没过 | **被否 / 一票否决** → 沉默归档 |
| 评估这个动作 | 四道门评估 | **过会评估 / 投决会评审** |

UI 按钮文案：`"进入四道门"` → `"上投决会"`；`"正在交给四道门…"` → `"正在上会…"`；web-admin badge `"四门通过"` → `"全票过会"`。

### 3. 纯概念外衣层，底层零改动

和 ADR 0005 完全一致——这是 UI / 文案 / 注释层的概念外衣，**底层数据结构、API、DB、事件名全部保持原样，无 DB migration**：

- 模块名 `gate`、类型 `GateDetail` / `GateFeedback`、`gate_evaluations` 表、`failed_gate` (1..4)、`g1_thickness` / `g2_anti_consensus` / `g3_window` / `g4_edge` 键、四个归档池
- API `POST /v1/gate/evaluate`、Go `Gate` interface、事件 `gate.evaluated`
- migration `004_create_gates_commitments` 冻结不动

### 4. Prompt 无需改

四位分析师的 Mastra prompt 早已是分析师口吻（引用 `ANALYSTS.*.name` / `ANALYSTS.*.question`），正文不含"四道门"字样。只更新 `mastra/src/agents/analysts.ts` 顶部登记表注释，点明"这一层即投决会"。

## Consequences

- **正面**：总称与"四位分析师"同源（分析师 = 委员，这一关 = 投委会）；动词体系完整且全是基金圈黑话（上会 / 过会 / 被否 / 一票否决），把原来"过门 / 没过门"的全部说法都接住；强化"给个人配机构级投委会"的叙事；零 migration。
- **代价**：一次性跨约 30 个活跃文件的文案 / 注释替换。
- **不动（历史快照）**：`docs/归档/**`、ADR 0005 本身、migration 004、`docs/原型/*.html`——它们记录的是当时的状态，不回改。
