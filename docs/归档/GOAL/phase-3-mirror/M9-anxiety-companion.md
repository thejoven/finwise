# M9 · 持仓陪伴 E4

> Phase 3 · W19-W20 · 2 周 · 行为指纹 + LLM 安慰师

---

## 上下文

M9 是 Phase 3 的开门——**当用户焦虑时, 把当初的判断递回他手里**。

E4 焦虑卡是产品最难做对的页面。
- 做粗了 → 像风险提示弹窗
- 做巧了 → 像贴心的私人助理
- 做错了 → 让用户更焦虑

正确的形态是**像一位主笔的按语, 引用用户自己 1 月 22 日签字的话, 帮他想起那时候的判断**。

- ✅ Phase 2 完成

## 目标

### 行为指纹
追踪用户打开承诺页的频次:
- 一天 1 次 → 平常
- 一天 3+ 次 → 可能焦虑
- 一天 5+ 次 → 强焦虑

数据存 Redis(24 小时窗口)+ events 表(永久记录)。

### Editor Agent
新建 Mastra Agent, 任务: 当用户打开承诺页 3+ 次时, 生成"主笔按语"——

> "今天的价格波动不触发你写过的任何退出条件。
> 那篇大V文章不在你的判据里。它不该改变你 1 月 22 日的判断。"

关键: 引用具体退出条件, 引用具体的"已经发生的事"。

### E4 焦虑陪伴页
用户进入承诺页时, 系统判断是否显示焦虑卡:
- 不焦虑 → 普通持仓页
- 焦虑 → E4 焦虑陪伴页(覆盖在持仓页上)

### Task 9.2 · Editor Agent prompt

参考 `产品文档/04_第四层_承诺执行.md` 的 "主笔按"。

核心约束:
- 不是"风险提示"语气, 是"主笔按语"语气
- 必须引用具体退出条件
- 必须引用用户原话(从 events 表)
- 不预测涨跌, 不评论市场
- 字数 50-100 字, 不超过 3 句

### Task 9.3 · Editor Workflow

输入: commitment_id + 当前打开次数 + 当前股价(mock)
输出: 一段 60-100 字的主笔按

**Phase 3 价格数据用 mock**, 不接真实行情 API。
只需要"一个数字"模拟波动场景, 让 Editor Agent 有内容可写。

### Task 9.5 · "我坚持要退出" 流程

不是按钮, 是 link 样式(弱化它):

```typescript
<Pressable onPress={openExitFlow}>
  <Serif size={12} italic style={{ color: theme.color.muted, textAlign: 'center' }}>
    我坚持要退出
  </Serif>
</Pressable>
```

点击 → 进入"重新评估"流程(不是一键退出, 是走严肃流程):
- 写 events 表 `companion.exit_insisted`
- 引导用户回答 3 个问题(M5 的简版)
- 如果用户坚持, 才真正改 commitment 状态为 CLOSED
- 不强行劝阻, 但流程让人慢下来

## 反模式特别警告

- ❌ 不要"⚠️ 风险提示" 红色 banner
- ❌ 不要弹"建议持仓"或"建议退出"
- ❌ 不要"AI 分析师认为" 措辞
- ❌ 不要展示股价 K 线图(违反"细节里有真相")
- ❌ 不要触感反馈(用户已在焦虑, 不要再刺激)
- ❌ 不要 Toast 提示"系统识别到你在焦虑"

正确形态:**像主笔在报刊版面上写的一段按语**, 安静、有判断、引用具体事实。
