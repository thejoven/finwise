# M7 · 承诺书生成 + PDF

> Phase 2 · W14-W15 · 2 周 · LLM 创作 + PDF 渲染

---

## 上下文

M7 把 M6 通过的信号 + 用户的 M5 追问数据, 编成一份**承诺书**。

承诺书是产品的物理化契约——不是建议, 不是分析, 是给未来某个想反悔的自己看的私人信件。

签字后, 这份承诺书会被渲染成 PDF 归档。

## 目标

### Narrator Agent
新建一个 Mastra Agent, 任务是生成承诺书的核心叙述。
不是 trade ideas、不是 analyst report, 是**"给未来的自己看的理由块"**。

- 报刊感顶栏
- 资产名(大字 italic)
- 仓位 / 持仓时长(JetBrains Mono)
- 退出条件(罗马数字列表)
- "你 6 个月后会感谢自己的理由"段落
- 签字按钮

## 任务列表(高层)

### Task 7.1 · Narrator Agent prompt

参考 `产品文档/04_第四层_承诺执行.md` 的承诺书模板。

prompt 核心约束:
- 不用分析师语气, 用"你"称呼用户
- 引用用户原话(从 M5 追问数据里)
- 不预测涨跌, 只说"已经发生的事的兑现"
- "你 6 个月后会感谢自己的理由" 段落必须包含 3 条真实回顾
- 字数控制 300-500 字
- 退出条件必须 ≥ 2 条, ≤ 4 条

### Task 7.3 · 承诺书 schema

```typescript
const CommitmentSchema = z.object({
  thesis_id: z.string().uuid(),
  asset: z.object({
    ticker: z.string(),
    name: z.string(),
  }),
  action: z.enum(['buy', 'sell', 'hold']),
  position_pct: z.number().min(0).max(100),    // 占可投资资产 %

  reasons_for_future_self: z.array(z.string()).min(3).max(5),
  generated_at: z.string().datetime(),
});
```

### Task 7.4 · PDF 渲染服务

Go + chromedp:

```go
func renderPDF(commitment Commitment) ([]byte, error) {
    html := renderTemplate("commitment.html", commitment)
    
    ctx, cancel := chromedp.NewContext(context.Background())
    defer cancel()
    
    var pdf []byte
    err := chromedp.Run(ctx,
        chromedp.Navigate(htmlDataURL(html)),
        chromedp.WaitVisible("body", chromedp.ByQuery),
        chromedp.ActionFunc(func(ctx context.Context) error {
            var err error
            pdf, _, err = page.PrintToPDF().
                WithPrintBackground(true).
                WithMarginTop(0.4).
                WithMarginBottom(0.4).
                WithPaperWidth(8.27).WithPaperHeight(11.69). // A4
                Do(ctx)
            return err
        }),
    )
    return pdf, err
}
```

PDF 模板 `templates/commitment.html`:
- 复用原型 v4 的报刊风 CSS
- 字体用 Google Fonts CDN(PDF 一次性渲染, 网络访问 OK)
- 包含: 标题、签字日期、四道门评估摘要、理由块、退出条件

## 反模式特别警告

- ❌ 不要"投资逻辑"段落(改为"理由块")
- ❌ 不要技术指标 / K 线 / 图表
- ❌ 不要风险披露免责声明(那是金融机构的事, 这是私人契约)
- ❌ 不要"AI 推荐"措辞("基于你的判断"不是"基于 AI 分析")
- ❌ PDF 不要水印 / 不要 footer 写 "Generated by AI"

正确形态:**像律师起草的、给用户自己看的、不会让用户觉得"这是 AI 写的"的契约**。

## 已知坑

1. **Narrator Agent 容易写成分析师风格**, prompt 必须有 anti-pattern examples
2. **chromedp 第一次启动慢**, 用连接池或预热
3. **PDF 字体加载**: 用 base64 inline 字体或 Google Fonts CDN
4. **"理由块"必须真引用**, 不能 LLM 编, 用 tool 拉历史 events
5. **签字按钮在 M8 实现**, M7 只做静态展示
