# 第三层 · 确定性评估

## 这一层是整个产品的良心所在

做对了，它是 conviction engine；做砸了，它就是又一个 AI 荐股。

## 最难的问题

> **AI 怎么知道"现在没有高确定性机会"，而不是为了显得有用，硬凑一个出来给你？**

这不是技术问题，是产品哲学问题。

绝大多数 AI 产品的失败都在这里——KPI 是 engagement，所以它们必须**总有话说**。但这个产品恰恰相反：

> **它的最高品质是大多数时候沉默。**

## 高确定性的操作定义

> **一个机会是高确定性的，当且仅当——即使它失败了，你回头看也会说"我当时的判断没错，是世界变了"，而不会说"我当时就该多想想"。**

这个定义把"高确定性"从**预测准不准**，挪到了**推理过不过得了关**。前者靠运气，后者可控。

巴菲特买可乐亏过钱，但他从不后悔买可乐——因为推理过得了关。这就是高确定性。

## 四位分析师：全员通过才提示，任一否决就沉默

> **命名说明**：早期版本把这一层叫"四道门"（门 1 / 门 2 / 门 3 / 门 4），太抽象、用户不好理解。
> 现在改成**四位各司其职的分析师**，对每一条信号轮流审核——更像一个真实的研究评审台。
> 四位的判断**全部由 LLM (Mastra agent) 给出**，不再是写死的阈值规则；只有在 Mastra 不可用时才回退到启发式兜底。

每位分析师只回答一个问题。四位全员放行，才提示用户；任何一位否决，立刻沉默归档。

### 佐证分析师 · 信号厚度（原门 1）
- **它问**：指向同一结论的独立证据够不够厚？
- **独立**：来自不同行业、不同人、不同场景
- **通过示例**：供应商涨价 · 同事抢机器 · 群里讨论瓶颈——三条独立信号都指向"内存稀缺"
- **否决示例**：只有一篇研报说看好，没有一线印证
- **怎么判**：LLM + RAG 召回你的历史信号，综合"单条信号自身厚度 + 跨信号主题广度"打分（单条够厚也能过，不再硬性要求 3 条）

### 共识分析师 · 反共识程度（原门 2）
- **它问**：市场是不是已经都知道、都定价了？
- **用什么测**：价格走势、研报覆盖、媒体声量
- **通过示例**：你看到的 HBM 稀缺，主流研报还在讨论 GPU 周期
- **否决示例**：所有人都在说 AI，标的已经涨了 80%
- **怎么判**：LLM 用叙事经济学 / 反身性 / base rate 给"叙事拥挤度"打 0-100 分，低于阈值算 leading view
- **指方向（2026-06 新增）**：判定"已被定价"时不再只是否决——额外指出**市场还没定价的相邻方向**（同资产的二阶 / 三阶角度，或受益链上下游环节）。"往哪看"，不是"买什么"：死路变线索，守住不荐股红线，不报估值 / 目标价 / 数字（那是受益链分析师的活）

### 时机分析师 · 时间窗口（原门 3）
- **它问**：现在出手是不是太早或太晚？还有多久这件事会被定价？
- **理想窗口**：大致几周到半年的前瞻窗口
- **通过示例**：下季度财报会暴露毛利变化，还有 2-3 个月窗口
- **否决示例**：催化剂已经发生 / 窗口太远（3 年以上）
- **怎么判**：LLM 判催化剂时序与前瞻窗口是否成立——比写死的"1-6 个月"更聪明：年度催化剂的长窗口也能过，已经落地的事件即使月数合规也不过

### 能力圈分析师 · 个人认知优势（最严格的一位，默认不通过）
- **它问**：你为什么有资格做这个判断？你凭什么比市场更懂这件事？
- **通过示例**：你是行业一线，信号直接来自工作，不是二手
- **否决示例**：你只看了新闻，没有圈层优势

## 能力圈分析师的反自我欺骗设计

人都喜欢相信"我懂这个"。所以能力圈分析师必须有一套**反自我欺骗的子测试**：

| 测试 | 通过条件 | 谁来判 |
|---|---|---|
| 能用一句话向不懂这个行业的朋友解释为什么这事会发生吗？ | 能（讲到根因 / enabling condition，而非复述结论） | LLM |
| 这条信号是你直接接触到的，还是别人转告的？ | 直接 | 元数据（有无 primary signal） |
| 过去 12 个月你在这个领域有过成功的预判吗？ | 至少 1 次 | 冷启动留空，第五层复盘后纳入 |
| 如果你错了，你能说出**具体哪个环节**会出问题吗？ | 能（给得出可证伪的退出条件） | LLM |

"能解释"和"知进退"这两项由 LLM 判断；"是否亲历"是元数据；"历史战绩"冷启动期留空。
能解释 ∧ 亲历 ∧ 知进退 都成立，能力圈分析师才放行。

**第四项尤其重要：真正的高确定性不是"我觉得会发生"，是"我知道它在什么条件下不会发生"。说不出失败条件的人，他的"自信"是错觉。**

## 评估的运行流程

四位分析师**并行审核**（彼此独立，互不依赖），任何一位否决即归档：

```
候选机会
   ↓
佐证分析师 ───── 否决 → 沉默, 证据不够厚, 进观察池
共识分析师 ───── 否决 → 沉默, 市场已定价, 进舍弃池 (附带"未被定价的方向")
时机分析师 ───── 否决 → 沉默, 窗口不合适, 进日历池
能力圈分析师 ─── 否决 → 沉默, 不是你的菜, 进教训池
   ↓
四位全员放行 · 提示用户
"我从你的数据里看到一个机会, 要不要展开聊?"
   ↓
进入第四层 · 承诺执行
```

## 这个流程最反常的设计：每一次否决都通往沉默

任何一位分析师否决时，系统**不是降级提示**（"虽然不够强，但可以考虑"），而是直接归档、沉默。

这是产品最难的克制。

绝大多数 AI 产品的设计思路是"用户付了钱，必须给他价值感"——所以即使是模糊机会也要包装成"参考"。但这个产品的世界观从一开始就是：

> **模糊机会是负价值**——它会消耗你的注意力和仓位，让你错过真正的高确定性机会。

这条产品哲学要写进 system prompt 的第一行：

> **当不确定时，沉默优于发声。模糊的机会不是低质量的高确定性机会，是另一种东西——是噪音。**

## 沉默不等于什么都不做：归档系统

每条被否决的信号要去哪？

### 佐证分析师否决的（证据不够厚）→ 观察池
继续监听。如果接下来一个月又有 2-3 条相关信号，自动重新评估。
**你之前内存那条线就是这样**——单独看每条信号都不够强，但累积起来就过线了。

### 共识分析师否决的（市场已定价）→ 舍弃池
系统会标注"这条你看到时已经晚了"，复盘时拿出来分析：**下次怎么早一个月看到？**
并且不止于"晚了"——共识分析师会附上**未被定价的方向**（往哪看的二阶角度 / 受益链相邻环节），让一条死路至少留下一个线索。

### 时机分析师否决的（窗口不对）→ 日历池
窗口太远的（比如 3 年后的事），设置自动提醒，到临近窗口时重新激活。

### 能力圈分析师否决的（不是你的菜）→ 教训池
最干脆的处理——这事确实在发生，但**不是你能赚的钱**，承认就好。
**这一池的存在本身就是产品的诚实：它在帮你识别什么不该做，而不是只告诉你该做什么。**

四个池加起来，是你的**外部记忆系统**。这才是订阅一年付费的真正价值——不是几条荐股，是这套**经过你校准的信号管道**。

## 分析师评审在内存案例上回放

| 时间 | 你的输入 | 系统状态 |
|---|---|---|
| 1月初 | "群里讨论 Mac Studio 抢购" | 佐证分析师暂未放行（单条）→ 入观察池 |
| 1月中 | "供应商说内存交付要等" | 佐证分析师放行（2 条独立信号佐证） |
| | （共识分析师审）| 主流研报仍聚焦 GPU → 放行 |
| | （时机分析师审）| 估算 3-6 个月窗口 → 放行 |
| 1月下 | 系统提示："你想聊聊这条吗？" | 能力圈分析师审 |
| | 你回答四问 | 一线工作信号、能解释清楚、能说出失败条件 → 放行 |
| | **四位全员放行** | **进入第四层，讨论仓位** |

整个过程**没有荐股**——它从头到尾在做的事，是**帮你把已经在你脑子里的东西，组织成一个可下注的形状**。
