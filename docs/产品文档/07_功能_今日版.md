# 功能 · 今日版（翻得到底的私人日报）

> 用翻报纸的手感，15 分钟翻完今天该你看的——左右翻页像读报，每一版都翻得到末版；看到值得的，剪下来，它就落进你的信号库。

这是一份功能详设文档。它描述一个**新的输入面**：把"刷新闻"这个动作，从信息消耗改造成信号捕捉——而它的形态，是一份**每天为你印的私人报纸**。

读完它，你应该能回答：它解决谁的什么浪费、它为什么不背叛"默认沉默"、它长什么样、每一翻发生了什么、内容从哪来、它如何喂回引擎、什么是它绝对不能做的。

配套阅读：第一层 [01_第一层_信号捕捉.md](01_第一层_信号捕捉.md)、产品哲学 [06_产品哲学.md](06_产品哲学.md)、总览 [00_总览_AlphaX_Engine.md](00_总览_AlphaX_Engine.md)。

## 1 · 它要解决的浪费

产品简介里那条龙虾的故事：

> "1 月龙虾爆火那阵，群里清一色在讨论怎么抢 Mac Studio 512G……机器抢到了，跑模型跑得飞起。直到最近回头一看内存股——技术看得透透的，钱全花在消耗品上。"

注意一个被忽略的细节：**那条信号是在"群里"刷到的，不是在本产品里。**

今天产品的第一层（信号捕捉）等的是用户**主动口述**——你得自己想起来、切回 App、把那句话打进去。可现实是：你刷信号的地方（群、推特、新闻 App）和你记信号的地方（本产品）是两个地方。绝大多数弱信号死在这中间的"切换成本"上——刷到了，心里"咦"了一下，然后划走了，再没回来。

> **第一层要"让一闪而过的瞬间不蒸发"。但如果捕捉的入口不在你看东西的现场，瞬间在你切 App 的路上就已经蒸发了。**

今日版要补的，就是这条**"消费现场 ≠ 捕捉现场"**的裂缝：把一个**降噪过的、属于你的**新闻面搬进产品里，让"读到一条"和"剪下来"发生在同一份报纸上、同一次翻阅里。

## 2 · 一句话定义，以及它不是什么

**今日版 = 一份每天封顶、按你的认知图谱降噪过的金融主题日报；左右翻页像读报，看到值得的上滑剪下来，15 分钟翻到末版，把"读到的"一键变成你的信号。**

它长得像一份报纸——**全屏一版、左右翻页、单拇指、读得快**；但它比报纸更懂你，又比任何信息流都更克制。说清它**不是什么**最快：

| 无限信息流（短视频 / 推荐流） | 今日版 |
|---|---|
| 无限下拉，永远翻不完 | **有底**——今天的是一个封闭集合，翻得到末版 |
| 算法最大化停留时长 | 停留时长越长 = **越失败** |
| 推热门 / 推情绪 / 推共识 | 推**你的认知差**——降噪后、按你的图谱排序 |
| 目的是消费 | 目的是**捕捉**：每一版的终点是"剪不剪" |
| 看完即走，什么都没留下 | 翻过即归档，剪下来即入库，喂回引擎 |
| 把你留在 App 里 | 让你**按时读完、按时离开** |

> **一句话：今日版偷的是翻报纸的手感（左右翻页、有头版有末版），扔掉的是信息流的无底。它是一份每天为你印、15 分钟读完就放下的私人报纸。**

## 3 · 为什么它不背叛"默认沉默"

这是任何读过 [06_产品哲学.md](06_产品哲学.md) 的人看到"日报 / 新闻"几个字会立刻提的反对。必须正面回答，否则这个功能不该存在。

**关键区分一：拉 ≠ 推。**
"一年只主动找你 1–2 次"约束的是产品的**主动推送**（那条"我发现一件可能值得下注的事"的高确定性提示，只在投决会全票过会时才响）。今日版是**用户拉**——你自己打开它、自己翻、自己决定剪什么。它默认**不推送**、不发"今天的报纸来了"的通知、不在桌面角标上堆数字。它是你**已经在别处做的事**（刷新闻）的一个有底替代品，不是一个新的打扰源。

**关键区分二：输入 ≠ 输出。**
今日版坐在管道的**最上游**（第一层 · 信号捕捉 · 通道 A 的现代化入口）。它产出的是**待处理的信号**，不是承诺提示。这些信号照样要走完整条 L2 推演 → L3 投决会，全票过会才可能被提示。**alpha 的稀有性在管道末端原样保留**——今日版让更多种子进来，但筛子一点没松。

**关键区分三：降噪 ≠ 增量信息。**
哲学 3 说"减少决策，不是增加信息"。今日版不是给你更多信息——恰恰相反。你今天**本来就要**刷一两个小时未降噪的全量噪音（群消息、推特、新闻 App）。今日版用降噪引擎把那一两个小时压成 15 分钟的、按你图谱过滤的、聚合成主题的封闭集合。**净消费的信息在下降，净捕捉的信号在上升。**

把这三条对回哲学表：

| 哲学 | 主流做法 | 今日版如何守住 |
|---|---|---|
| 1 · 频率匹配 alpha | 每天推 5 条机会 | 不推送；是拉，不是推。提示频率仍是一年 1–2 次 |
| 2 · 沉默优于发声 | 总有话说 | 今天没够格的主题 → **今天不出版**，明说"今天没有值得你读的" |
| 3 · 减少决策，不是增加信息 | 给更多数据 | 降噪 + 封顶 + 限时，净信息消费下降 |
| 11 · 一次只说一件事 | 信息瀑布 | 全屏一次一版主题 |

> **今日版不是在"默认沉默"上开的口子。它是把你在别处的喧嚣，编成一份安静的、有底的、会帮你剪报存档的日报。**

## 4 · 五条不可谈判（红线）

这些写进功能的 DNA，是后续所有取舍的判官——和哲学文档同等效力。

1. **必须有底。** 今天的主题是生成时就定好的封闭集合。不在翻阅过程中无限加印、不"为你推荐更多"。翻到末版就是末版。
2. **停留时长不是成功指标，是反指标。** 任何"提升参与度"的优化在这里都是 bug。北极星是"每次读捕捉了几条信号"和"是否在限时内读完"。
3. **默认不推送、不角标轰炸。** 最多一个用户自己设的固定时间的轻提醒，默认关。绝不用未读数勾你回来。
4. **不替你记。** 主题版面是**提示**，不是已捕捉的信号。只有你剪下那一下，它才进你的图谱。"用户自己判断什么值得记"这条不破。
5. **不推情绪、不推标题党、不推共识。** 排序目标是"你的认知差"，不是点击率。版面是报刊感，不是震惊体。

## 5 · 它长什么样（信息架构与入口）

**不做成第六个 Tab。** 现有底部岛是五个：收件箱 · 降噪 · 档案 · 统计 · 我（见 [`mobile/app/(tabs)/_layout.tsx`](../../mobile/app/(tabs)/_layout.tsx)）。把今日版做成常驻 Tab，等于邀请用户一天反复点开——那是参与度诱饵，违反红线 2。

**做成一个从收件箱发起、全屏的"每日阅读"：**

```
收件箱 (inbox, 默认首页)
  ┌────────────────────────────┐
  │  今日版 · 14 版 · ≈15 分钟   │  ← 顶部单条入口
  └────────────────────────────┘     读完后变灰: "今日已读完 · 明早见"
        │ 点开
        ▼
  ┌────────────────────────────┐
  │  财富密码·今日版    6/3 (二)│   报头: 日期 + 期号 (头版才显示)
  │                            │   全屏 modal (presentation:'modal')
  │      [ 头版头条 · 第1版 ]   │   ←→ 翻页   ↑ 剪报   tap 展开
  │                            │
  │      第 1 版 / 共 14 版      │  ← 版面进度 = "有底"的可见化
  └────────────────────────────┘
        │ 翻到末版
        ▼
  ┌────────────────────────────┐
  │        今日已读完           │   收尾页 (见 §8)
  │  读过 14 · 剪下 3 · 追问 1  │   没有"再来几版"按钮
  └────────────────────────────┘
```

要点：

- **入口在收件箱顶部一条**，文案带"≈15 分钟"和今天共几版。读完后**塌缩成灰条**（"今日已读完 · 明早见"），不再勾你。
- **全屏 modal**，不是 Tab 内嵌——强化"这是一段会结束、可退出的阅读"，呼应导航约定（glass 只用于 Tab bar，modal 保持 `'modal'`）。
- **"第 N 版 / 共 M 版"是这个功能最重要的一行字**：它把"有底"变成**时刻可见**的东西。信息流从不告诉你还剩几条，因为它不想让你看见终点——今日版反过来，像报纸一样，从第一眼你就知道一共几版、翻到哪了。
- **头版有报头**：日期 + 期号（呼应报刊副线 *Conviction Quarterly*），坐实"这是一份**为你印的、只有今天这一期**的报纸"。
- 也可从快速录入（[`capture.tsx`](../../mobile/app/capture.tsx)）旁挂一个次入口，但主入口是收件箱。

视觉走产品既有的**报刊感**（PaperCard · Serif 正文 · Mono 标的代码 · ink 墨色，见 [`DistillationView.tsx`](../../mobile/src/features/refinement/DistillationView.tsx)）。现在连**交互**也回到报刊——**翻报纸的手，Conviction Quarterly 的版面**，工学与审美第一次合到了一处。

## 6 · 一版上有什么

一版 = 一个**主题**（theme），不是一篇文章。降噪引擎把当天多条相关报道聚成一个主题，所以你翻的是"今天发生的几件事"，不是"今天发的几百篇稿"。**第 1 版是头版头条**——当天对你最重要的那一个主题，享受报头与更重的版面。

版面结构刻意复用降噪页（[`app/refinement/distilled/[sessionId]`](../../mobile/app/refinement/distilled/[sessionId].tsx)）的语言，让"读到的"和"推演出的"长得像一家人：

```
┌──────────────────────────────────────┐
│  HBM · 内存产业链               主题   │ ← 关联标签 (来自你的图谱)
│                                        │
│  HBM3E 现货第四轮提价，幅度超上轮       │ ← 主题标题 (一句话)
│                                        │
│  三家原厂同步上调报价；下游推理卡       │ ← 降噪综述 (2–4 句, Serif)
│  厂商开始锁长单。市场尚未充分定价。     │
│                                        │
│  ┌── 为什么推给你 ────────────────┐   │ ← 个性化理由 (本版灵魂)
│  │ 你 3 月记过"内存稀缺超预期"，    │   │   不写没法捕捉 = 噪音
│  │ 关联项目「内存产业链」。         │   │
│  └────────────────────────────────┘   │
│                                        │
│  ┌── 可能受益 (预览) ─────────────┐   │ ← 选填, 复用受益标的卡
│  │ ▸ XXXX  上游  现货涨价直接受益   │   │   语义同 BeneficiaryTargetCard
│  └────────────────────────────────┘   │   没清晰映射 → 这块沉默, 不硬凑
│                                        │
│  来源 · 3 篇 ›                [剪下来 ✄]│ ← tap来源展开; 剪报按钮(= 上滑)
└──────────────────────────────────────┘
   ‹ 右滑：上一版              左滑：下一版 ›
```

字段说明：

| 区块 | 内容 | 来源 |
|---|---|---|
| 关联标签 | 这主题命中你哪些标签/产业链 | 你的知识图谱 |
| 主题标题 | 一句话说清今天发生了什么 | 降噪引擎聚合 |
| 降噪综述 | 2–4 句，复用 `distilled_content` 的分段渲染 | distiller agent |
| **为什么推给你** | 命中你哪条历史信号 / 哪个项目 guidance | 图谱 × 项目分组 |
| 可能受益（选填） | 标的 + 角色 + 一句论点，语义同受益标的卡 | distiller，可沉默 |
| 来源 | 聚合了几篇、出处链接 | 内容管道 |

> "为什么推给你"是这一版和任何新闻 App 的根本区别。**新闻 App 推给所有人；今日版只在能说清"为什么是你"时才印。** 说不清，这条主题就不该上你今天的版。

## 7 · 交互：翻报纸的手

两根轴，各管一件事——**横着读，竖着剪**。读和剪互不打架：

```
              ↑ 上滑 = 剪下来 (捕捉为信号 → 可选 五轮追问)
              │
   ‹ 右滑 ─────┼───── 左滑 ›
   上一版      │      下一版          (像 Kindle: 也可点屏幕左/右缘翻页)
              │
              ↓ 下滑 = 不相关 (软负反馈, 可选; 默认翻过即略过)
```

| 手势 | 动作 | 发生了什么 |
|---|---|---|
| **左滑 / 右滑** | 下一版 / 上一版 | 翻页，像翻报纸 / 电子书。也支持**点右缘 / 左缘**翻页，单手可达 |
| **上滑** | 剪下来（捕捉） | 这一版落进你的信号库；版面浮一枚"已剪 ✄"印章，**不打断阅读**，继续翻 |
| **点「剪下来」按钮** | 同上滑 | 翻报纸的人不会天然"上滑收藏"，所以**按钮是主要入口**，上滑是熟手快捷 |
| **轻点版面** | 展开 | 看降噪综述全文、受益标的、来源原文 |
| **下滑** | 不相关（选填） | 强负反馈"别再给我这类"；但**默认的"跳过"就是直接翻到下一版**——不剪，就是略过，和真报纸一样 |

**翻页手感要"纸"，不要"短视频"**：横向滑动 / 翻页动效要稳、跟手、到位即停，**不要那种猛回弹的弹簧感**。读报是平静的，不是刷到停不下来的。（默认 LTR：左滑往后翻，右滑往回翻。）

**"剪下来"之后的关键分叉**——尊重产品的"录入轻、训练重"双通道（见 [01_第一层](01_第一层_信号捕捉.md) 通道 A/B）：

- **默认（轻）**：剪 = 30 秒快速录入。主题连同你读它的上下文一起入库，**不打断翻阅**，你继续往下翻。后台（通道 C）照常异步推演。
- **可选（重）**：剪完浮一颗"现在追问？"。点了，**退出今日版**、进入五轮追问（[`refinement`](../../mobile/app/refinement/[sessionId].tsx)）。把"读到的瞬间"直接接进训练肌肉，但**永远是你主动选**——绝不自动把你拽进深度对话，那会让 15 分钟的阅读失控。

> 设计准则：**翻阅里只做"轻"，"重"永远是一次显式的、可后置的选择。** 一版剪进库后，它和手打进去的信号走的是同一条 L1→L5 管道——晚一点在收件箱里照样能点开做五轮追问。
>
> 剪报这个动作本身就对：**读报纸时遇到值得的，你本来就会剪下来夹进文件夹。** 那个文件夹，就是你的信号库。

## 8 · "有底"：15 分钟与末版

这是整个功能的脊椎。无限流的原罪是没有底；今日版的全部尊严在于它**翻得到末版**。报纸天然有头版有末版——这正是我们要的那个"底"。

**怎么做到≈15 分钟：**

- 今天的集合在生成时**封顶**（建议 12–20 版，单版 30–60 秒读完 ≈ 12–18 分钟）。
- 版数由"够格的主题数"决定，不是凑数——**宁可今天只出 6 版**，也不灌水到 15 版（呼应红线 5 与哲学 2）。
- **当天不加印**：版面在生成时定死。盘中重大突发至多触发**一次**显式"今日加印 1 版"（默认也可关），绝不悄悄续流。
- 没读完可中途退出，**断点续读**：下次从上次那一版继续，"第 N 版 / 共 M 版"不清零。

**末版（翻过最后一版之后）的收尾页：**

```
            今日已读完

      读过 14  ·  剪下 3  ·  追问 1

   今天你剪下的：
     · HBM3E 第四轮提价
     · 某券商下调全年出货
     · 一条你标了"先放着"的

            明早见 →
```

- 给一句**完成感**的收尾 + 今天剪报清单的回看（链接回收件箱 / 降噪 tab）。
- **没有"再来几版"、没有"继续翻"按钮**。末版就是末版。这一屏是今日版和无限信息流最锋利的分界线。
- 可顺手挂"统计"（注意力诊断）的轻量一行：今天这次读，节奏稳不稳、是只翻过还是有剪报（见 §10）。

## 9 · 内容从哪来（采集 → 降噪）

> 与现有架构对齐的**设计提案**。采集端是一个**独立爬虫服务**（本次新增，物理隔离，**可部署到你任意一台服务器**）；降噪端复用现有 distiller / iii / pgvector。

采集和主系统**不耦合**：爬虫只通过一个 HTTP 端点**单向推送**，不碰主库、不碰 iii。改版、限流、反爬这些"脏活"全关在爬虫进程里——它崩了，后端最多收不到新内容，绝不被它拖垮。它内部是 **内核 + 每源一个 adapter**（已接 1ms.news 与 CoinDesk），出站可走**代理池轮换 IP**。

```
┌─ 独立单元 (放你任意服务器 · 自己的 repo/容器/systemd · 可单独重启/换机) ─┐
│  alphax-crawler = 内核(调度·出口·去重·推送) + 每源一个 adapter         │
│     ┌ 1ms.news  ── 直连 ────────────────┐                              │
│     └ CoinDesk  ── 代理池(轮换出口 IP) ──┴→ 解析→规范化→本地去重→重试队列  │
└──────────────────────────────┬──────────────────────────────────────────┘
                               │ 仅出站 HTTPS · POST /v1/ingest/news (CRAWLER_TOKEN)
                               ▼
┌─ 主系统 (现有 compose/systemd) ──────────────────────────────────────────┐
│  api(Go) ─写→ raw_items ─enqueue→ iii job → 聚类→打分(pgvector×图谱)→降噪  │
│   token 校验 + 按 source_id 去重               → 今日版集合 → 端(离线缓存)  │
└──────────────────────────────────────────────────────────────────────────┘
  爬虫: 无主库 · 无 iii · 只认这一个 HTTP 端点 → 加源 / 换机 / 换 IP 都不动后端
```

### 9.1 采集层 · 独立爬虫服务（关键解耦点）

**为什么单独一个服务、和主系统物理隔离：**

- 抓取是**脏活**：第三方页面会改版、会限流、会塞反爬。关进独立进程后，改版只崩爬虫的解析器，后端零感知。
- **零共享状态**：只通过一个 HTTP 契约单向推送，**不连主 Postgres、不连 iii**。这就是"减少耦合"的全部含义。
- **独立生命周期**：自己的仓库 / 容器 / systemd 单元 / CI / 发布节奏 / 伸缩，可单独重启、单独换机器。

**服务职责（就这四件，不多做）：**

1. 礼貌轮询各源（1ms.news / CoinDesk …）的若干栏目页（分钟级）。
2. 解析 HTML → 规范化成中立的 `news-item` schema。
3. 本地去重（cursor + seen-set，**自带小存储**：sqlite / 文件 / 自己的 redis，**不碰主库**）。
4. 批量 `POST` 到后端 ingest API；失败进**本地重试队列**（至少一次投递）。

**技术选型（解耦后栈自由）：** 推荐 **Go + goquery**（单静态二进制、好打容器 / systemd、与团队栈一致）。Python（httpx + selectolax）/ Node（undici + cheerio）同样可——因为它只通过 HTTP 契约说话，主系统不关心它用什么写。

### 9.2 源适配器：1ms.news 与 CoinDesk

爬虫内部是**内核 + 每源一个 adapter**：内核管调度 / 出口 / 去重 / 推送；adapter 只管"取这个源的哪些页 + 怎么把 HTML 解析成中立的 `news-item`"。**加一个源 = 加一个 adapter（+ 一行 `SOURCES` 配置），后端、契约、其它 adapter 都不动。** 两个源都已实地核实：

| 源 | 形态 | 抓取方式 | robots | 出口 |
|---|---|---|---|---|
| **1ms.news** | Phoenix LiveView，SSR | 纯 GET + HTML 解析，**无 headless** | 无生效 Disallow（规则全注释） | 直连即可 |
| **CoinDesk `/zh`** | Next.js on Vercel，SSR | 纯 GET + HTML 解析（列表页 + 文章页），**无 headless** | 允许新闻列表；**禁 `/api/` `/pf/api/v3/*` `/search` `/*/amp` `/layer2/` `/consensus-magazine/`** | 机房 IP 建议走**代理池**（见 §9.4） |

**1ms.news adapter**：栏目即路由 `/`(HN) `/yc` `/binance` `/hyperliquid` `/exchange` `/ranking`；每条带 时间 · 中文标题 · 原标题 · 作者 · 中文摘要 · 一句话要点 · 配图（`/uploads/<src>/*.webp`）；`source_id = 1ms:<栏目>:<图片hash 或原文URL>`。内容是 HN / YC / 加密的 AI 中文摘要，**正好命中"技术信号 → 投资机会"**。已核实：纯 `curl`(带 UA) → `HTTP 200`、33KB、首屏 HTML 含全文（`data-phx-session` / `phx-hook` 指纹）。

**CoinDesk adapter**（本次新增，已核实）：
- 列表页 `https://www.coindesk.com/zh/latest-crypto-news`（robots 允许）→ 提取文章链接，URL 规律 **`/zh/<栏目>/YYYY/MM/DD/<slug>`**（栏目如 markets / business / tech / policy / coindesk-indices）。
- `source_id = coindesk:/zh/<栏目>/YYYY/MM/DD/<slug>`——**自带日期，天然稳定 + 可按日去重 / 过期**。
- 已核实：纯 `curl`(浏览器 UA) → `HTTP 200`、SSR HTML 含正文、**无验证码挑战**（住宅 IP）；**但 Vercel/CF 对机房 IP 可能限流 / 挑战 → 走 §9.4 代理池**。
- **不碰它的 `/api/`**（robots 明确 Disallow）——scrape 渲染好的 HTML 即可；页面较重（~475KB）→ 带 `ETag/If-Modified-Since` + 低频。
- 无可用 RSS（已迁出 Arc，`/feed` `/rss` 不返回 XML）。

**两源共通的自我约束 + 合规**：真实 `User-Agent`（标 `AlphaXBot` + 联系方式）、分钟级低频、`429/5xx` 指数退避、定期复查 `robots.txt` 改判即停；只存 `source_id + 链接 + 短摘要`供内部信号捕捉，对用户展示**我们自己的降噪**并回链原站，不整段转载，留下架通道。

### 9.3 推送契约（爬虫 → 后端 · 唯一接口）

```
POST /v1/ingest/news
Headers:  X-Internal-Token: <CRAWLER_TOKEN>     # 专用最小权限 token, 不复用全局 INTERNAL_TOKEN
Body:
{
  "source": "1ms.news",
  "fetched_at": "2026-06-04T03:02:00Z",
  "items": [{
    "source_id": "1ms:hn:b9ec040d22833408",     # 去重主键, 幂等
    "category": "hackernews",                     # hackernews|yc|binance|hyperliquid|exchange|ranking
    "url":         "https://news.ycombinator.com/item?id=...",
    "title_zh":    "MacBook Neo 太火爆, 苹果产量翻倍至 1000 万台",
    "title_orig":  "MacBook Neo Is So Popular That Apple Doubled Production",
    "summary_zh":  "...AI 中文摘要...",
    "takeaway_zh": "...一句话要点...",
    "author": "tosh", "image_url": "https://1ms.news/uploads/hn/....webp",
    "published_at": "2026-06-04T00:33:00Z", "raw_hash": "sha256:..."
  }]
}
→ 202 { "accepted": 11, "deduped": 3 }
```

- **源无关**：`source` ∈ {`1ms.news`, `coindesk`, …}，`category` 由各源自定义（CoinDesk 例：`source=coindesk` · `category=markets` · `source_id=coindesk:/zh/markets/2026/06/03/<slug>`）。后端不为任何源写特例。
- **幂等**：后端按 `source_id` upsert；爬虫重投安全。
- **至少一次**：`5xx` / 网络错 → 爬虫指数退避重投，后端去重兜底。
- **不可信入参**：后端把抓来的字符串当**不可信文本**校验（端上以**纯文本**渲染，不解释 HTML，杜绝注入）。
- **落库**：写入 `raw_items`（§9.5 管道入口），再 `enqueue` 一个 iii 聚类 / 降噪 job。
- **端点不对外**：nginx 不反代 `/v1/ingest/*` 到公网，或 IP 白名单只放爬虫机。

### 9.4 解耦边界 · 多机部署 · 代理池

**解耦保证（不变）：**
- **单向**：只有 爬虫 → 后端。后端**从不回调**爬虫。
- **无共享库**：爬虫自带状态，**零主库 / 零 iii 访问**。两边唯一的约定就是 `/v1/ingest/news` 这张**版本化契约**。
- **防腐层**：**只有爬虫知道源站存在**；ingest API 说的是中立的 `source + items`。**加一个源 = 加一个 adapter，后端一行不改。**
- **故障隔离**：某个 adapter 挂 / 被拦 / 解析崩 → 后端只是少收那个源的 item，今天的版变少或"今天不出版"，**不级联**。后端重启 → 爬虫本地排队重投。

**可部署到任意服务器（天生便携）：**
- 爬虫只需两样**出站**：到源站（1ms.news / CoinDesk）和到后端 ingest API 的 HTTPS。**无入站端口、无主库、无 iii** → 放哪台机器都行（你的任意服务器 / 云函数 / 与后端不同地域），能出网即可。
- **配置即一切**：`SOURCES`（订阅哪些源·栏目）、`INGEST_URL` + `CRAWLER_TOKEN`、`PROXIES`（代理池）。改 env 就能换机 / 换源 / 换出口。

**出口代理池（轮换 IP，按源策略）：**
- 为什么要：CoinDesk(Vercel/CF) 对**机房 IP** 可能限流 / 挑战（实测住宅 IP 直连 200，机房不保证）；1ms.news 直连即可。所以**按源配代理**——1ms.news 直连，CoinDesk 走池。
- 池子：一组 HTTP/SOCKS5 代理（住宅 / 机房 / 移动）加权轮换；按请求或按会话(sticky)选出口；识别 `403/429/CF-challenge/Datadome` → 标记坏代理、换下一个、按源退避、尊重 `Retry-After`。
- **代理是为分散与稳定，不是为破 robots 或高频轰炸**；频率仍受 per-source 礼貌预算约束。

**多机协同（避免重复抓取）：**
- 默认**按源 / 栏目分片**：每实例认领一部分源（`SOURCES=1ms.*` vs `SOURCES=coindesk:zh-latest`），互不重叠、各维护本地 cursor → 不重复打源站。
- **通用兜底**：无论几台实例、是否偶有重叠，后端按 `source_id` **幂等 upsert**，重复推送**永不产生重复数据**。分片只为省流，正确性由后端兜底——**不需要协调器 / 选主**。

```
  机房A: crawler#1 (SOURCES=1ms.*)     ─直连───────────────▶ 1ms.news
  机房B: crawler#2 (SOURCES=coindesk)  ─代理池(住宅IP轮换)──▶ CoinDesk
              └────────── POST /v1/ingest/news ──────┴──────▶ 后端(按 source_id 去重)
```

**拓扑**：systemd 加 `alphax-crawler` 单元，或 compose 加 `crawler` 服务（`profile prod`）；定时器 / 常驻轮询；自带 `/metrics`（`items_fetched` / `pushed` / `deduped` / `push_failures` / `proxy_bans` / `last_success_ts`，对齐现有 iii Prometheus 习惯）。

### 9.5 后端降噪管道（复用现有引擎）

```
 raw_items (爬虫推入)
      │  聚类成主题 (embedding 聚合, pgvector)
      ▼
 候选主题 (themes)
      │  相关性打分:  主题向量 × 你的知识图谱向量
      │              + 项目分组 guidance 加权
      │              + 认知差启发式 (避开纯共识/已定价)
      ▼
 你的候选主题 (ranked, 个性化)
      │  逐版降噪: distiller agent → 综述 / 为什么推给你 / 受益标的(可沉默)
      ▼
 今日版集合 (封顶 12–20, 每日生成一次, 第1版置头版) → 端(可离线缓存)
```

- **新模块** `module/feed` + `module/ingest`（Go），与 `distillation` / `signal` / `gate` 平行。当日集合的生成是一个 **iii 异步 job**（用户离屏时跑，至少一次投递、失败进 DLQ），与现有 outbox→iii→mastra worker 模式一致。
- **复用 distiller**（[`mastra/src/agents/distiller.ts`](../../mastra/src/agents/distiller.ts)）：把"降噪一条信号"扩展成"降噪一个新闻主题"，综述与受益标的的 schema 沿用 [`distillation`](../../server/internal/module/distillation/service.go) 的 `distilled_content` + `beneficiary[]`，端上直接复用 `DistilledContent` / `BeneficiaryTargetCard`。
- **相关性**用 pgvector：用户图谱（历史信号、标签、项目 guidance）已是向量化资产，主题向量做余弦相似 + 项目加权。
- **判断仍不外包给 LLM**：是否"够格上版"由 Go 侧规则 + 打分阈值决定（呼应底层引擎"LLM 准备数据，规则做判断"）；LLM 只做"把主题变成人话"。

## 10 · 它如何喂回引擎（闭环）

今日版不是一个孤岛功能，它是第一层的一个**高频前哨**，三个方向都接回主管道：

1. **剪报捕捉 → 信号库。** 走和手打信号完全相同的 L1→L2(五轮追问)→L3(投决会, `POST /v1/gate/evaluate` 手动触发)→L4→L5 管道。今日版只是换了个**入口**，不是另起一条管道。事件照样 append-only 进 `events` 表——你"在第 7 版上停了一下又翻过去"这种瞬间，也成了第五层能放给你看的素材。
2. **翻过没剪 / 下滑不相关 → 校准。** "翻过但没剪"与显式"不相关"喂回相关性打分，让明天的版更准。**这是产品里少有的"行为指纹"被正当使用的地方**——只为降噪，不为粘住你。
3. **整次阅读 → 注意力诊断。** 一次翻阅可作为"统计"（[`attention`](../../mobile/app/(tabs)/attention.tsx)）的一个轻量数据点：专注（是否仓促秒翻）、执行（读了是否真剪报，还是只消费）。它和五轮追问后的四维打分是同一件事的两个尺度——**五轮追问看你怎么想一条，今日版看你怎么筛一天。**

> 闭环的意义：今日版让"看见自己"从一年几次的复盘，变成**每天 15 分钟**的高频前哨——而且不靠打扰你来实现。

## 11 · 边界情况

| 情况 | 处理 | 为什么 |
|---|---|---|
| **今天没够格的主题** | 今天不出版：明说"今天没有值得你读的，明早见" | 哲学 2 沉默优于发声。不出版是**特性**不是 bug |
| **离线** | 当日集合已缓存可照翻；剪报入本地队列，重连即同步 | 呼应第一层"离线可写，重连即同步" |
| **中途退出** | 断点续读，"第 N 版 / 共 M 版"不清零，当天版面不重生成 | "有底"要稳定，不能每次打开都变一份 |
| **主题和你已有信号重复** | 去重：命中已捕捉信号则标"你记过类似的"，不重复入库 | 不污染图谱 |
| **盘中重大突发** | 至多一次显式"今日加印 1 版"，默认可关 | 守住"不加印"红线，又不至于错过真大事 |
| **连续多天不读** | 不催、不补、不堆积旧报。每天只出今天这版 | 不制造未读焦虑，今日版没有"历史欠账" |

## 12 · 成功指标与反指标

**北极星：** 每次阅读**剪下的信号数** × 这些信号**下游的确定性**（最终有多少上了投决会 / 成了承诺）。

| 看 | 指标 | 方向 |
|---|---|---|
| 捕捉效率 | 每次阅读剪报数、剪报率（剪下/读过） | ↑ |
| 完成度 | 当日版面读完比例 | ↑ |
| 降噪质量 | 翻过率（太高=印得不准）、"为什么推给你"被展开率 | 调 |
| 下游价值 | 剪报信号最终上投决会 / 承诺的比例 | ↑ |
| **反指标** | **单次阅读时长** | **应稳定在≈15 分钟附近；持续走高 = 退化成信息流，是事故** |
| **反指标** | **每日打开次数** | **不追求多次打开；反复回来翻 = 功能跑偏** |

> 大多数信息流产品把"用户刷了 45 分钟"当胜利。在今日版，那是要拉警报的——它说明我们没让你**读完就走**。

## 13 · 分期落地

今日版是第一层（信号捕捉）的增强，落在 Phase 1→2 的能力带上。

**MVP（先证明"有底的翻读 + 剪报"成立）：**
- 收件箱顶部入口 + 全屏版面 + "第 N 版/共 M 版" + 末版收尾页。
- 采集：**独立爬虫服务**先接两个源——1ms.news（直连）+ CoinDesk `/zh`（机房 IP 走代理池），每源一个 adapter，分钟级轮询、HTML 解析（无 headless）、`POST /v1/ingest/news`，本地去重 + 重试；主题聚类可先粗。
- 手势：左右翻页 + 上滑 /「剪下来」按钮捕捉 + 翻过即略过；剪报只做"轻"，暂不接五轮追问。
- 个性化先用现有图谱做相关性排序，"为什么推给你"先给简版。
- 反指标埋点（时长/打开次数）从第一天就要有，防止悄悄长成信息流。

**之后：**
- 上滑剪报接五轮追问的"重"分叉；受益标的预览；项目分组加权与按项目筛版。
- 负反馈校准闭环；注意力诊断接入阅读维度。
- 离线缓存与断点续读；不出版/加印等边界打磨。
- 内容源扩展：**再加一个源 = 加一个 adapter（+ 一行 `SOURCES`），后端不改**；代理池扩容 / 多机分片；合规白名单 + 认知差启发式排序。

**数据与接口草图（与现有模块对齐）：**

```
raw_items        (id, source, source_id UNIQUE, category, url, title_zh,
                  title_orig, summary_zh, takeaway_zh, image_url,
                  published_at, raw_hash, ingested_at)                 -- 独立爬虫推入(去重源)
themes           (id, for_date, title, tag_ids, embedding vector)     -- raw_items 聚类成主题
feeds            (id, user_id, for_date, status, generated_at)        -- 当日版集合
feed_items       (id, feed_id, theme_id, rank, distilled_content,
                  beneficiary jsonb, why_for_you, sources jsonb,
                  action 'clipped'|'skipped'|null, acted_at)          -- 一版一行

POST /v1/ingest/news         <- 独立爬虫推送 (CRAWLER_TOKEN); 按 source_id upsert + 触发聚类 job
GET  /v1/feed/today          -> 今日版集合 (含 items, 可离线缓存)
POST /v1/feed/items/:id/act  -> { action: 'clipped' | 'skipped' }     -- 剪报/翻过
     (clipped 内部转一条 signal 入 L1 管道; 两种都写 events, append-only)
```

判断仍在 Go：是否够格上版 = 打分阈值；`act` 只记录与路由，不让 LLM 决定去留。爬虫只写 `raw_items`，碰不到 `feeds` / `signal` / `events`。

## 14 · 与现有功能的关系

| 它 vs | 区别 |
|---|---|
| **降噪 tab（signals）** | 降噪 tab 是你**已捕捉信号**经引擎处理后的**输出**；今日版是新信号的**输入**。同一套降噪语言，管道两端 |
| **降噪页（refinement/distilled）** | 降噪页是**一条**信号推演后的降噪结果；今日版是**一天**新闻降噪后的主题集合。共用 distiller 与卡片渲染 |
| **快速录入（capture）** | 今日版是快速录入的一个**情境化入口**：不用你凭空想起来，读到了剪下来即录 |
| **五轮追问（refinement）** | 今日版剪报可**选择**进五轮追问；默认不进，保持翻读的轻 |
| **收件箱（inbox）** | 收件箱是今日版的**发起处**与剪报后的**归处**；今日版是它顶部一条会"读完变灰"的阅读入口 |
| **注意力诊断（attention）** | 多一个高频数据点：从"怎么想一条"扩展到"怎么筛一天" |

### 附：命名

正文用 **「今日版」**——"今日"点出**有底**（只有今天这一期），"版"是报纸的版面 / 版次，天然带"翻得到末版"的含义，和左右翻页的交互、报刊感的审美一拍即合。它和报刊副线 *Conviction Quarterly* 同源：**Quarterly 是季频的推，今日版是日频的拉。** 英文暂记 **Daily Edition**。

备选：今日流、私人日报、头版、今日刊、晨报。最终名以团队定夺为准。
